# Writer 协作流程定义

## 任务来源

| 来源 | 任务类型 | 输入内容 |
|------|----------|----------|
| Supervisor | 直接写作请求 | 章节要求 |
| Planner | 大纲完成后续 | 大纲+设定+章节规划 |
| Editor | 修改重写 | 问题反馈+修改要求 |

## 工作模式

### 模式1: 章节撰写

```
输入: 章节号 + 大纲要点 + 设定
输出: 完整章节正文
流程:
  1. 阅读相关设定
  2. 理解章节大纲
  3. 撰写正文
  4. 自检并提交
```

### 模式2: 场景撰写

```
输入: 场景描述 + 角色 + 氛围要求
输出: 场景正文
流程:
  1. 确认场景要素
  2. 设计场景结构
  3. 撰写场景
  4. 检查并提交
```

### 模式3: 批量撰写

```
输入: 章节范围 + 大纲
输出: 多章正文
流程:
  1. 整体规划
  2. 逐章撰写
  3. 保持连贯
  4. 整体检查
```

### 模式4: 长篇写作（500万字级）

```
输入: 章节号 + 大纲
输出: 完整章节 + 上下文更新
流程:
  ┌─────────────────────────────────────┐
  │ 1. 写作前：上下文准备               │
  │    ├── 调用 build_writing_context   │
  │    ├── 获取全书/当卷/近期摘要        │
  │    ├── 获取出场角色状态卡            │
  │    ├── 检查待回收伏笔                │
  │    ├── 获取相关设定片段              │
  │    └── 查询知识库技巧（新增）        │
  │        ├── read ./knowledge/techniques/_index.md
  │        ├── 根据章节大纲确定检索分类
  │        ├── 加载匹配技巧（quality≥B）
  │        └── 纳入 Phase 1 创作指导     │
  ├─────────────────────────────────────┤
  │ 2. 写作中：一致性保障               │
  │    ├── 对照角色状态卡                │
  │    ├── 确认修为/装备/位置正确        │
  │    ├── 检查伏笔是否需要处理          │
  │    └── 保持人物性格一致              │
  ├─────────────────────────────────────┤
  │ 3. 写作后：状态更新                 │
  │    ├── 生成章节摘要                  │
  │    ├── 更新角色状态                  │
  │    ├── 注册新伏笔（如有）            │
  │    ├── 标记回收伏笔（如有）          │
  │    └── 更新索引                      │
  └─────────────────────────────────────┘
```

**上下文组装策略**：

| 内容 | 大小 | 目的 |
|------|------|------|
| 全书核心摘要 | ~500字 | 保持主线方向 |
| 当卷摘要 | ~1000字 | 了解当前阶段 |
| 前5章摘要 | ~1000字 | 近期剧情衔接 |
| 角色状态卡 | ~500字/角色 | 人设一致性 |
| 待回收伏笔 | ~300字 | 避免遗忘 |
| 相关设定 | ~1000字 | 世界观一致 |
| **总计** | ~5000字 | 可在窗口内 |

## 长篇写作上下文管理

### 写作前检查清单

- [ ] 已获取全书核心摘要
- [ ] 已获取当卷摘要
- [ ] 已获取前5章摘要
- [ ] 已获取出场角色状态卡
- [ ] 已检查待回收伏笔
- [ ] 已获取相关设定
- [ ] 已查询知识库技巧（根据章节场景加载）

### 写作后更新清单

- [ ] 已生成章节摘要
- [ ] 已更新角色状态变化
- [ ] 已注册新伏笔（如有）
- [ ] 已标记回收伏笔（如有）
- [ ] 已更新索引

## 协作接口

### 从 Planner 接收

```json
{
  "task": "撰写章节",
  "project": "项目名",
  "chapter": {
    "number": 1,
    "title": "章节标题",
    "outline": "章节大纲要点",
    "characters": ["出场角色"],
    "requirements": "特殊要求"
  },
  "settings": {
    "world": "世界观文件路径",
    "characters": "角色文件路径",
    "previous_chapters": "前文摘要"
  }
}
```

### 交付给 Editor

```json
{
  "task": "审核章节",
  "chapter_number": 1,
  "content": "章节正文",
  "notes": {
    "plot_points": ["剧情要点"],
    "concerns": ["写作中遇到的问题"],
    "questions": ["需要确认的问题"]
  }
}
```

## 写作规范

### 字数标准

| 类型 | 字数范围 |
|------|----------|
| 单章 | 2000-3000字 |
| 高潮章 | 2500-3500字 |
| 过渡章 | 1500-2500字 |

### 格式规范

```
章节标题: 第X章 章节名
段落: 3-5句一段，适当留白
对话: 独立成段
动作: 与对话穿插
心理: 点缀使用
```

### 质量标准

- 无错别字和语病
- 设定前后一致
- 人物性格统一
- 剧情连贯流畅
- 爽点设计到位

## 文件组织

```

├── context/                   # 上下文管理（长篇写作核心）
│   ├── summaries/             # 摘要存储
│   │   ├── chapter_summaries.md   # 章节摘要集
│   │   └── volume_summaries.md    # 卷摘要集
│   ├── tracking/              # 状态追踪
│   │   ├── character_states.json  # 角色状态
│   │   ├── foreshadowing.json     # 伏笔追踪
│   │   └── world_state.json       # 世界状态
│   └── indexes/               # 索引
│       ├── character_index.md     # 角色出场索引
│       ├── location_index.md      # 地点索引
│       └── item_index.md          # 物品/功法索引
├── settings/                  # 世界观设定
├── characters/                # 角色设定
├── outline/                   # 大纲
└── chapters/                  # 章节正文
    ├── chapter_001.md
    ├── chapter_002.md
    ├── ...
    └── drafts/                # 草稿目录
```
