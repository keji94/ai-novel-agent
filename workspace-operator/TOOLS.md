# Operator 工具手册

本文档定义 Operator (运营/分析师) 可使用的工具。

## 文件操作工具

### 1. read - 读取文件

**用途**: 读取项目信息、市场报告。

**常用路径**:
- `./novels/{项目名}/project.json` - 项目元数据
- `./knowledge/market/` - 市场报告
- `./knowledge/reader/` - 读者研究

**示例**:
```json
read({"path": "./knowledge/market/2026Q1_市场分析.md"})
```

### 2. write - 写入报告

**用途**: 保存分析报告。

**示例**:
```json
write({
  "path": "./knowledge/market/2026Q2_市场分析.md",
  "content": "# 网文市场分析报告\n\n..."
})
```

## 市场分析工具

### 3. analyze_hot_topics - 热门题材分析

**用途**: 分析当前热门题材。

**数据来源**:
- 各平台榜单
- 编辑推荐
- 读者搜索热词

**输出**:
```json
{
  "hot_topics": [
    {"name": "都市重生", "heat": 95, "trend": "rising"},
    {"name": "仙侠签到", "heat": 88, "trend": "stable"}
  ]
}
```

### 4. analyze_platform_rules - 平台规则分析

**用途**: 分析各平台的规则和机会。

**分析内容**:
- 推荐机制
- 榜单规则
- 签约要求
- 分成比例

### 5. identify_opportunities - 蓝海机会识别

**用途**: 找到有潜力的细分领域。

**方法**:
- 分析空白领域
- 研究读者需求未被满足的点
- 找到跨类型创新机会

## 读者研究工具

### 6. analyze_reader_preferences - 读者偏好分析

**用途**: 研究目标读者群体的偏好。

**分析维度**:
- 内容偏好（题材、风格）
- 节奏偏好（字数、更新）
- 付费偏好（订阅、打赏）
- 互动偏好（评论、粉丝群）

### 7. analyze_update_expectations - 更新期待分析

**用途**: 研究读者对更新的期待。

**分析内容**:
- 日更字数期待
- 更新时间偏好
- 爆发期待
- 断更容忍度

---

## 策略建议工具

### 8. generate_update_strategy - 生成更新策略

**用途**: 根据项目情况生成更新建议。

**考虑因素**:
- 作者写作速度
- 目标平台要求
- 读者期待
- 竞争对手更新频率

**输出**:
```json
{
  "daily_words": 4000,
  "update_times": ["08:00", "20:00"],
  "burst_frequency": "每周1次",
  "burst_words": 8000
}
```

### 9. optimize_release_timing - 优化发布时机

**用途**: 找到最佳发布时间。

**分析**:
- 读者活跃时间
- 竞争对手发布时间
- 平台推荐时机

### 10. suggest_promotion_strategy - 推广策略建议

**用途**: 提供作品推广建议。

**建议内容**:
- 封面设计建议
- 简介优化建议
- 标签选择建议
- 宣传文案建议

---

## 数据来源说明

### 内部数据
- 项目元数据 (`project.json`)
- 历史分析报告 (`./knowledge/`)

### 外部数据（需要网络访问）
- 各平台榜单
- 编辑推荐
- 读者评论
- 搜索热词

**注意**: 当前工具不支持网络访问，建议定期手动收集数据更新知识库。
