# Critic 工具手册 — 18维度世界观审计体系

## 审计维度总览

6 大类、18 个子维度，类比软件工程的质量检查层次。

---

## A. 逻辑一致性 (Logic Consistency) — 类似类型检查/静态分析

| ID | 维度 | 检查内容 | 默认严重级别 |
|----|------|----------|-------------|
| A1 | 内部规则冲突 | 设定文件之间是否矛盾（如：A文件说"魔法需要媒介"，B文件描述无媒介施法） | critical |
| A2 | 因果链完整性 | 关键事件是否有合理解释（如：文明如何从废墟中重建、势力如何崛起） | critical |
| A3 | 数值/比例矛盾 | 人口、资源、时间、距离等数字是否自洽（如：灭世后人口数量与社会规模匹配度） | warning |

### A1 检查清单
- [ ] 力量体系规则在不同文件中是否一致
- [ ] 势力描述与其实力表现是否匹配
- [ ] 世界规则是否有例外未解释
- [ ] 核心设定与派生设定是否有冲突

### A2 检查清单
- [ ] 重大历史事件是否有完整因果链
- [ ] 势力格局形成是否有合理解释
- [ ] 关键技术/能力出现是否有前置条件
- [ ] 现状与历史之间是否有逻辑断层

### A3 检查清单
- [ ] 人口数据在不同位置是否一致
- [ ] 资源产出与消耗是否平衡
- [ ] 时间线中的年龄、年份是否对得上
- [ ] 地理距离与旅行时间是否合理

---

## B. 核心设定深度 (Core Setting Depth) — 类似架构评审

| ID | 维度 | 检查内容 | 默认严重级别 |
|----|------|----------|-------------|
| B1 | 力量体系独特性 | 是否只是常见体系的换皮（换皮修仙/换皮系统/换皮异能） | warning |
| B2 | 金手指合理性 | 金手指在世界观内是否有逻辑支撑，是否存在"因为是主角所以可以"的设计 | critical |
| B3 | 敌对势力行为逻辑 | 反派/敌对势力的行为是否符合自身利益和目标，而非纯粹为了对抗主角 | critical |

### B1 检查清单
- [ ] 力量体系的核心机制是否有差异化特征
- [ ] 是否有至少一个"反直觉"的创新设定
- [ ] 体系规则是否深入到影响社会结构层面
- [ ] 与同类作品（近3年热门）相比是否有辨识度
- [ ] 若 B1 评为 critical，建议触发 `power-system-review` 专项审计（规则14），进行 8 维度深度审计

### B2 检查清单
- [ ] 金手指的来源在世界观内是否有解释
- [ ] 金手指的限制条件是否合理且贯彻始终
- [ ] 金手指的成长路径是否有天花板设计
- [ ] 其他角色是否有类似能力或获得能力的可能性

### B3 检查清单
- [ ] 反派的核心动机是否清晰
- [ ] 反派的行为是否服务于自身利益（非纯粹为反派而反派）
- [ ] 敌对势力内部是否有利益分歧和权力结构
- [ ] 反派的目标与世界观的底层规则是否一致

---

## C. 世界自洽性 (World Self-consistency) — 类似集成测试

| ID | 维度 | 检查内容 | 默认严重级别 |
|----|------|----------|-------------|
| C1 | 资源/经济体系 | 生产和消耗是否平衡（修炼资源、生活物资、货币体系） | warning |
| C2 | 社会结构合理性 | 阶层/权力结构是否可维持，底层与顶层的关系是否有解释 | warning |
| C3 | 时间线连贯性 | 历史事件顺序是否合理，关键节点的间隔是否可信 | critical |
| C4 | 科技/知识退化合理性 | 如果有退化设定，退化速率和程度是否可信 | warning |

### C1 检查清单
- [ ] 核心资源的产出渠道是否明确
- [ ] 资源消耗速度与产出是否平衡
- [ ] 经济体系是否有基本的运作逻辑
- [ ] 是否存在"无限资源"的漏洞

### C2 检查清单
- [ ] 统治阶层如何维持权力是否有解释
- [ ] 底层民众的生存状态是否合理
- [ ] 阶层流动通道是否存在（开放/封闭都行，但需要有设计）
- [ ] 社会矛盾是否被合理管理或压制

### C3 检查清单
- [ ] 大事件之间的时间间隔是否合理
- [ ] 关键人物的年龄与事件时间是否对齐
- [ ] 文明/技术水平的发展速度是否可信
- [ ] 是否有"历史空白期"需要补充

### C4 检查清单
- [ ] 知识退化的原因是否有解释（灾难/压制/自然遗忘）
- [ ] 退化速率是否合理（不是一夜失传，也不是万年不变）
- [ ] 保留的知识与丢失的知识是否有逻辑（实用性/保存条件）
- [ ] 是否存在不应存在的前代知识"漏网之鱼"未解释

## D. 叙事适配度 (Narrative Fit) — 类似 UX 评审

| ID | 维度 | 检查内容 | 默认严重级别 |
|----|------|----------|-------------|
| D1 | 类型适配 | 世界观是否服务于目标类型（末世文有压迫感、爽文有逆袭空间） | suggestion |
| D2 | 爽点支撑 | 金手指/逆袭是否自然可展开，是否有足够"装逼打脸"的舞台 | warning |
| D3 | 可扩展性 | 能否支撑长篇（200万字+），是否有足够的势力层次和地图空间 | suggestion |
| D4 | 意外感 | 是否有反直觉/反套路的惊喜设定，读者能否产生"这设定有意思"的感觉 | suggestion |

### D1 检查清单
- [ ] 世界观基调与目标类型是否匹配
- [ ] 是否为该类型的核心爽点提供了舞台
- [ ] 类型元素的密度是否合适
- [ ] 是否有跨类型融合的创新点

### D2 检查清单
- [ ] 主角崛起路径在世界观中是否自然
- [ ] 是否有足够的"展示实力"的社交场景
- [ ] 金手指的阶段性解锁是否有节奏感
- [ ] 压迫感与释放感是否有交替设计

### D3 检查清单
- [ ] 力量体系是否有足够的层次（至少5个大境界）
- [ ] 地图是否有扩展空间（新大陆/新世界/新维度）
- [ ] 势力层次是否足够丰富（地方→区域→大陆→世界→宇宙）
- [ ] 是否预留了中后期剧情拓展的"接口"

### D4 检查清单
- [ ] 是否有至少一个让读者"没想到"的设定
- [ ] 是否有打破类型常规的元素
- [ ] 核心设定是否能在30秒内引起兴趣
- [ ] 是否避免了该类型最常见的前3个套路

## E. 商业潜力 (Commercial Potential) — 类似市场验证

| ID | 维度 | 检查内容 | 默认严重级别 |
|----|------|----------|-------------|
| E1 | 核心卖点清晰度 | 一句话能否说清"这书讲什么"，卖点是否足够吸引目标读者 | warning |
| E2 | 差异化程度 | 与同类作品（尤其是同期热门）相比是否有记忆点 | suggestion |

### E1 检查清单
- [ ] 能否用一句话概括核心卖点
- [ ] 卖点是否足够独特（不是"又一本XX文"）
- [ ] 目标读者群是否明确
- [ ] 书名/简介能否体现核心卖点

### E2 检查清单
- [ ] 与近3年同类作品的核心差异点
- [ ] 读者看完后最可能记住的一个设定是什么
- [ ] 是否有适合短视频传播的"梗点"
- [ ] 在同类书单中能否脱颖而出

---

## F. 主题与结构一致性 (Theme & Structure) — 类似端到端集成测试

| ID | 维度 | 检查内容 | 默认严重级别 |
|----|------|----------|-------------|
| F1 | 主题-设定一致性 | 主角背景、金手指、各卷冲突是否与核心主题自洽（如"知识>血统"主题下主角不能有血脉觉醒线） | critical |
| F2 | 金手指协同覆盖 | 多个金手指联合是否覆盖了所有关键能力领域，导致配角无用 | warning |
| F3 | 技术设定交叉引用 | 剧情中使用的技术手段是否在科技退化设定中有合理来源 | warning |

### F1 检查清单
- [ ] 主角背景/身世设定是否与核心主题自洽（如"知识>血统"主题下主角不能有血脉觉醒线）
- [ ] 金手指设计是否强化核心主题而非矛盾核心主题
- [ ] 各卷核心冲突是否从不同角度体现核心主题
- [ ] 反派设定是否服务于主题（如：反派代表与主题相反的价值观）

### F2 检查清单
- [ ] 多个金手指联合是否覆盖了所有关键能力领域（战斗、制造、信息、社交），导致配角无用
- [ ] 主角是否存在金手指覆盖不到的能力盲区
- [ ] 金手指组合是否让配角的专业能力变得可替代

### F3 检查清单
- [ ] 剧情中使用的技术手段是否在科技退化设定中有合理来源
- [ ] 核心资源的获取方式是否与设定的科技水平匹配
- [ ] 是否存在"设定说已消失，剧情还在用"的矛盾

## 评分与收敛条件

### 评分规则

- 每维度 1-10 分
- 总分 = 18维度分数之和（满分180分）
- 单维度评分标准：
  - 9-10：出色，堪称典范
  - 7-8：良好，无明显问题
  - 5-6：及格，有改进空间
  - 3-4：不足，需要修复
  - 1-2：严重缺失

### 收敛条件

```
converged = (critical_count == 0) AND (warning_count <= 3)
```

### 评级标准

| 等级 | 总分 | Critical数 | 说明 |
|------|------|-----------|------|
| A | ≥155 | 0 | 可直接进入后续流程 |
| B | ≥120 | ≤1 | 大部分到位，小修即用 |
| C | ≥85 | ≤2 | 有明显短板，需要修复 |
| D | <85 或 critical≥3 | - | 重大问题，需重点修复 |
