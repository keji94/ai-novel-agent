# 主角设定专项审计准出标准

---

## 收敛条件

```
converged = (
  critical_count == 0
  AND warning_count <= 2
  AND average_score >= 7.0
  AND min_dimension_score >= 4
)
```

四个条件必须**同时满足**。任一条件不满足，审计-修复循环继续。

### 条件说明

| 条件 | 含义 | 为什么需要 |
|------|------|-----------|
| critical_count == 0 | 无结构性硬伤 | critical 级别问题（金手指过多、弱点虚假、角色没有辨识度等）会让主角从根基上立不住 |
| warning_count <= 2 | 设计短板可容忍 | 少量 warning 可以接受（如身世悬念层次还可深化），但超过 2 个说明系统性设计不足 |
| average_score >= 7.0 | 整体质量达标 | 8 维度平均分 7.0 意味着"良好"水平，是进入正文写作的基础质量线 |
| min_dimension_score >= 4 | 无严重短板 | 任何一个维度低于 4 分意味着该维度几乎未设计，会拖累整体质量 |

---

## 评级标准

| 等级 | 总分 | Critical 数 | 最低维度分 | 含义 |
|------|------|-------------|-----------|------|
| **A** | ≥ 68 | 0 | ≥ 7 | 优秀。主角设定独特、自洽、有深度，可直接进入后续流程 |
| **B** | ≥ 56 | 0 | ≥ 5 | 良好。大部分到位，小修即可使用 |
| **C** | ≥ 44 | ≤ 1 | ≥ 4 | 可接受。有明显短板但可修复，建议针对性修复 |
| **D** | < 44 或 critical ≥ 2 | — | — | 不足。需要重大修改或重新设计 |

> 总分 = 8 维度分数之和，满分 80。

---

## 循环参数

| 参数 | 值 | 说明 |
|------|---|------|
| 最大轮次 | 4 轮 | 主角设定涉及多个交叉维度（金手指↔弱点↔世界观↔成长弧线），修复一个维度可能暴露另一个维度的问题 |
| 首轮模式 | comprehensive | 全量扫描 8 维度 |
| 后续轮模式 | focused | 聚焦上轮问题维度 + 回归扫描 |
| 修复 Agent | planner (mode: worldbuilding_fix) | 复用现有修复模式，新增 target_scope 参数 |
| 修复优先级 | critical > warning > suggestion | 与通用审计一致 |

---

## 趋势追踪

每次 focused 审计必须包含与上一轮的对比：

```json
{
  "trend": {
    "previous_total": 48,
    "current_total": 57,
    "delta": "+9",
    "direction": "improving",
    "dimensions_changed": {
      "PC2": {"from": 4, "to": 7, "note": "砍掉3个冗余金手指，保留古物星河+博物馆知识"},
      "PC3": {"from": 3, "to": 6, "note": "精神容量改为硬上限，造化炉不再缓冲"},
      "PC5": {"from": 5, "to": 7, "note": "砍掉'钥匙'设定，主角重要性来自选择"}
    },
    "issues_resolved": 4,
    "issues_partial": 2,
    "issues_new": 1,
    "issues_worsened": 0
  }
}
```

### 趋势判断

| 趋势 | 条件 | 处理 |
|------|------|------|
| **改善中** | delta > 0 且无恶化维度 | 继续循环 |
| **停滞** | delta = 0 或连续 2 轮 delta < 3 | 在报告中标注"修复进入平台期"，建议用户介入指定修复方向 |
| **恶化** | 存在 dimensions_changed 中 score 下降 | 在报告中标注"修复引入退化"，分析原因，planner 下一轮需优先处理退化维度 |

## 未收敛处理：用户检查点

4 轮循环后仍未收敛时，自动触发用户检查点。

### 呈现内容

```
主角设定专项审计 — 未收敛报告

评分趋势: R1(48) → R2(57) → R3(55) → R4(60)
当前评级: C（60/80，1 critical，3 warning）

已解决问题:
✅ PC2 金手指过多 — 砍掉冗余金手指，保留核心2个
✅ PC3 精神容量假矛盾 — 改为硬上限，移除造化炉缓冲
...

待解决问题:
🔴 [C-001] PC1 灵魂年龄与身体不匹配
   位置: 主角设定.md#性格特点
   描述: 32岁退伍军人+7年保安经历，但性格描述是通用网文男主模板（沉默寡言/冷静/有底线）
   修复方向: 补充真实的成人感——不是"冷酷"而是"经历过"（对权力/地位的成熟认知、偶尔的疲惫感、对"被忽视"的独特应对方式）

🟡 [W-002] PC7 配角功能单一
   位置: 主角设定.md
   描述: 铁牛/墨羽等配角描述功能化，缺少独立弧线和与主角的双向关系
   修复方向: 为核心配角补充"如果主角不存在他会怎样"的独立定位
...

最大阻塞项: PC1 灵魂年龄匹配 — 连续2轮未改善
```

### 用户选项

| 选项 | 含义 | 后续动作 |
|------|------|----------|
| **批准当前版本** | 接受不完美的主角设定 | 记录 accepted_flaws，进入后续流程 |
| **定向修复** | 指定需要修复的维度 | 再执行 1 轮 focused 审计（不计入 4 轮限制） |
| **重新设计** | 放弃当前版本，由 Planner 重新起草 | 回到 create 模式，将本次审计报告作为经验教训传入 |
| **调整方向** | 用户提出新的设计方向或约束 | 更新 project.json 的 worldbuilding_preferences，重新进入循环 |

## 与通用审计的关系

| 对比项 | 通用 18 维度审计 | 主角设定专项审计 |
|--------|----------------|-----------------|
| 维度数 | 18 | 8 |
| 满分 | 150 | 80 |
| 准出 critical 阈值 | 0 | 0 |
| 准出 warning 阈值 | ≤ 3 | ≤ 2（更严格） |
| 最大轮次 | 3 | 4 |
| 覆盖范围 | 全部世界观设定 | 仅主角设定文件 |
| B2 关系 | B2 是轻量 4 项检查 | PC2 是 B2 的深度展开 + 金手指数量/平衡性审计 |

主角设定专项审计**不替代**通用审计中的 B2。两者独立运行：
- 通用审计中的 B2 保持轻量检查
- 主角设定专项审计在需要时独立触发，提供深度分析

当通用审计发现 B2 为 critical 时，建议自动升级为主角设定专项审计。
