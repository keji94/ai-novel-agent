# 力量/修炼体系专项审计准出标准

---

## 收敛条件

```
converged = (
  critical_count == 0
  AND warning_count <= 2
  AND average_score >= 7.0
  AND min_dimension_score >= 4
)
```

四个条件必须**同时满足**。任一条件不满足，审计-修复循环继续。

### 条件说明

| 条件 | 含义 | 为什么需要 |
|------|------|-----------|
| critical_count == 0 | 无逻辑硬伤 | critical 级别问题（核心规则缺失、金手指无根基、代价机制空白等）是结构性缺陷，必须修复 |
| warning_count <= 2 | 设计短板可容忍 | 少量 warning 可以接受（如独特性细节还可深化），但超过 2 个说明系统性设计不足 |
| average_score >= 7.0 | 整体质量达标 | 8 维度平均分 7.0 意味着"良好"水平，是进入正文写作的基础质量线 |
| min_dimension_score >= 4 | 无严重短板 | 任何一个维度低于 4 分意味着该维度几乎未设计，会拖累整体质量 |

---

## 评级标准

| 等级 | 总分 | Critical 数 | 最低维度分 | 含义 |
|------|------|-------------|-----------|------|
| **A** | ≥ 68 | 0 | ≥ 7 | 优秀。力量体系设计成熟，可直接进入后续流程 |
| **B** | ≥ 56 | 0 | ≥ 5 | 良好。大部分到位，小修即可使用 |
| **C** | ≥ 44 | ≤ 1 | ≥ 4 | 可接受。有明显短板但可修复，建议针对性修复 |
| **D** | < 44 或 critical ≥ 2 | — | — | 不足。需要重大修改或重新设计 |

> 总分 = 8 维度分数之和，满分 80。

---

## 循环参数

| 参数 | 值 | 说明 |
|------|---|------|
| 最大轮次 | 4 轮 | 力量体系涉及多维度交叉（规则↔成长↔代价↔社会影响），修复一个维度可能暴露另一个维度的问题 |
| 首轮模式 | comprehensive | 全量扫描 8 维度 |
| 后续轮模式 | focused | 聚焦上轮问题维度 + 回归扫描 |
| 修复 Agent | planner (mode: worldbuilding_fix) | 复用现有修复模式，新增 target_scope 参数 |
| 修复优先级 | critical > warning > suggestion | 与通用审计一致 |

---

## 趋势追踪

每次 focused 审计必须包含与上一轮的对比：

```json
{
  "trend": {
    "previous_total": 52,
    "current_total": 61,
    "delta": "+9",
    "direction": "improving",
    "dimensions_changed": {
      "PS1": {"from": 5, "to": 7, "note": "核心概念机制补充完整"},
      "PS3": {"from": 4, "to": 6, "note": "补充了代价递进设计"},
      "PS5": {"from": 6, "to": 5, "note": "金手指解释部分退化，需关注"}
    },
    "issues_resolved": 5,
    "issues_partial": 2,
    "issues_new": 1,
    "issues_worsened": 0
  }
}
```

### 趋势判断

| 趋势 | 条件 | 处理 |
|------|------|------|
| **改善中** | delta > 0 且无恶化维度 | 继续循环 |
| **停滞** | delta = 0 或连续 2 轮 delta < 3 | 在报告中标注"修复进入平台期"，建议用户介入指定修复方向 |
| **恶化** | 存在 dimensions_changed 中 score 下降 | 在报告中标注"修复引入退化"，分析原因，planner 下一轮需优先处理退化维度 |

## 未收敛处理：用户检查点

4 轮循环后仍未收敛时，自动触发用户检查点。

### 呈现内容

```
力量体系专项审计 — 未收敛报告

评分趋势: R1(52) → R2(61) → R3(58) → R4(63)
当前评级: C（63/80，1 critical，4 warning）

已解决问题:
✅ PS1 核心规则清晰度 — 机制补充完整
✅ PS2 成长路径设计 — 等级定义明确
...

待解决问题:
🔴 [C-001] PS3 代价机制空白
   位置: 古物共鸣体系.md#核心规则
   描述: 体系中缺少使用力量的代价描述，精神侵蚀只在"发现史"中提到，核心规则部分未体现
   修复方向: 在核心规则中补充代价条款，定义代价的递进层次

🟡 [W-002] PS4 数据过度精确
   位置: 古物共鸣体系.md#回响类型
   描述: "破军型~40%"等精确百分比在末世120年背景下不合理
   修复方向: 改为粗略范围或标注"长安城文心派估算值"
...

最大阻塞项: PS3 代价与限制 — 连续2轮未改善
```

### 用户选项

| 选项 | 含义 | 后续动作 |
|------|------|----------|
| **批准当前版本** | 接受不完美的力量体系 | 记录 accepted_flaws，进入后续流程 |
| **定向修复** | 指定需要修复的维度 | 再执行 1 轮 focused 审计（不计入 4 轮限制） |
| **重新设计** | 放弃当前版本，由 Planner 重新起草 | 回到 create 模式，将本次审计报告作为经验教训传入 |
| **调整方向** | 用户提出新的设计方向或约束 | 更新 project.json 的 worldbuilding_preferences，重新进入循环 |

## 与通用审计的关系

| 对比项 | 通用 15 维度审计 | 力量体系专项审计 |
|--------|----------------|-----------------|
| 维度数 | 15 | 8 |
| 满分 | 150 | 80 |
| 准出 critical 阈值 | 0 | 0 |
| 准出 warning 阈值 | ≤ 3 | ≤ 2（更严格） |
| 最大轮次 | 3 | 4 |
| 覆盖范围 | 全部世界观设定 | 仅力量体系设定文件 |
| B1 关系 | B1 是轻量 4 项检查 | 8 维度是 B1 的深度展开 |

力量体系专项审计**不替代**通用审计中的 B1。两者独立运行：
- 通用审计中的 B1 保持轻量检查
- 力量体系专项审计在需要时独立触发，提供深度分析

当通用审计发现 B1 为 critical 时，建议自动升级为力量体系专项审计。
