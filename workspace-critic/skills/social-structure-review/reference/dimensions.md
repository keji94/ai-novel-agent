# 社会结构专项审计维度规范

本文件定义社会结构专项审计的 8 个维度、43 个检查项。独立于通用 18 维度体系，仅在社会结构专项审计时激活。

## I. 结构逻辑 (Structural Logic)

> 类比：建筑的承重结构——如果权力和经济的基础逻辑不成立，上层的一切设定都会坍塌。

### SS1: 权力结构逻辑

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 这个权力结构为什么能存在？凭什么不被人推翻？ |

**检查清单**：

- [ ] **SS1-1 权力来源一致性**：统治阶级的权力来源是否清晰定义，且与世界观底层规则一致。例如：靠武力统治的世界，统治者必须拥有最强的武力或控制最强武力的手段；靠知识垄断统治的世界，必须有阻止知识传播的可行机制。权力来源不能是"因为剧情需要"。

- [ ] **SS1-2 压制机制逻辑**：防止下层挑战权力的机制是否逻辑成立。检查：下层为什么无法反抗？是因为力量差距？信息不对称？还是制度性的分化策略？压制机制必须是世界观内可解释的，不能依赖"下层人不够聪明/不够勇敢"。

- [ ] **SS1-3 权力更替规则**：权力更替（继承、政变、改革、选举）是否有明确的规则或合理的历史路径。即使设定中"从未发生过权力更替"，也必须解释为什么——是什么机制阻止了更替的发生。

- [ ] **SS1-4 稳定性校准**：权力结构的稳定性/脆弱性是否与故事需求匹配。过于稳定的社会无法产生冲突驱动力；过于脆弱的社会无法解释"为什么至今还没崩溃"。校准点：设定的稳定性应该刚好能维持现状，但存在明确的崩溃路径。

- [ ] **SS1-5 结构性张力**：是否存在至少一个可驱动剧情的结构性张力或裂缝。这个裂缝不是"表面矛盾"，而是权力结构本身的内在矛盾——比如：统治者需要觉醒者维持生存，但觉醒者的力量威胁统治（觉醒者悖论）。结构性张力是推动剧情的根本动力。

- [ ] **SS1-6 暴力分配合理性**：军事、警察、暴力力量的分配是否合理。谁掌握武装？武装力量对统治者的忠诚度基于什么？是否存在武装力量倒戈的可能性？武装力量的规模与人口、资源的比例是否合理？

### SS2: 经济可持续性

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 这个社会的人靠什么活着？经济循环能否闭环？ |

**检查清单**：

- [ ] **SS2-1 生产活动识别**：核心生产活动和产出是否明确（粮食、材料、能源、手工业品）。不能只写"有农业""有商业"，必须具体到：谁在生产、生产什么、用什么生产、产量大致多少。在资源受限的末世/异世界设定中，这一点尤为关键。

- [ ] **SS2-2 资源流动链条**：从生产到消费的资源流动是否有合理链条。例如：粮食从农田 → 收割 → 储存 → 分配 → 消费，每一步由谁负责？链条中哪个环节最脆弱？是否存在断链风险？不能有"凭空出现"的物资。

- [ ] **SS2-3 货币背书逻辑**：货币/交换媒介是否有明确的背书或采用逻辑。贵金属货币需要贵金属来源；以物易物需要双方认可等价物；信用货币需要发行者的信用基础。不能"大家突然都接受了某种货币"而没有解释。

- [ ] **SS2-4 贸易机制定义**：社会层级或区域间的贸易是否有定义的机制。内城和三环外之间怎么交易？不同聚集地之间是否有贸易（如果设定说没有，为什么没有——是因为封锁还是因为不需要）？贸易的利润流向哪里？

- [ ] **SS2-5 经济-权力一致性**：经济稀缺/充裕是否与权力结构一致。资源极度匮乏的社会，统治阶层对资源的控制力应该更强（因为控制资源就是控制生存）。如果设定中资源匮乏但统治阶层对经济没有控制力，这是矛盾。

- [ ] **SS2-6 冲突压力点**：经济系统是否有至少一个可驱动冲突的压力点。资源争夺、分配不公、贸易垄断、货币崩溃——经济矛盾是社会冲突最根本的驱动力之一。如果经济系统完美运转、没有任何压力点，社会冲突就缺乏底层动力。

## II. 势力动态 (Faction Dynamics)

> 类比：建筑内部的结构墙——势力格局决定了冲突的空间布局和剧情的交互网络。

### SS3: 势力内部结构

| 属性 | 值 |
|------|---|
| 严重级别 | warning |
| 核心问题 | 每个势力不是一块铁板——它内部怎么运作？ |

**检查清单**：

- [ ] **SS3-1 内部层级识别**：每个命名势力是否有可识别的内部层级或决策机制。不能只写"公会由高阶共鸣者组成"——必须有：谁说了算？怎么决策？内部有没有派系？成员怎么晋升？至少核心势力需要有内部结构描述。

- [ ] **SS3-2 内部派系分裂**：至少 2 个主要势力有可见的内部派系或分裂。内部矛盾是势力行为的深层驱动力——如果一个势力永远团结一致，它要么太简单，要么是工具人势力。内部派系可以是：改革派 vs 保守派、激进派 vs 温和派、利益集团 A vs 利益集团 B。

- [ ] **SS3-3 招募/准入标准**：势力成员/招募标准是否定义或暗示。怎么加入？有没有门槛？是血缘制、师徒制、考试制、还是实力说话？准入标准直接影响社会流动性设定。

- [ ] **SS3-4 领导层交接机制**：领导层继承或交接机制是否存在。如果存在：是什么机制？如果不存在：为什么——是制度设计缺陷、还是刻意安排、还是权力高度集中导致"没有交接的必要"？交接机制的缺失本身可以是重大剧情冲突点。

- [ ] **SS3-5 组织文化独特性**：势力是否有影响其行为的独特组织文化。不同势力的行事风格应该有差异——同样是"追求力量"，武道派和匠造派的组织文化应该完全不同。组织文化影响势力对同一事件的不同反应。

### SS4: 跨势力关系

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 势力之间的关系网络是否有逻辑？还是各说各话？ |

**检查清单**：

- [ ] **SS4-1 关系映射完整性**：主要势力间的联盟/敌对/中立关系是否明确映射。最好有清晰的关系图（至少在设定者的脑中）。每对势力之间的关系状态应该是明确的，不能模糊。

- [ ] **SS4-2 关系动机基础**：每个势力间关系是否有清晰的动机基础（利益交换、意识形态、历史恩怨、外部威胁）。联盟不能是"因为都是好人"，敌对不能是"因为都是坏人"。每个关系背后必须有具体的利益计算或历史渊源。

- [ ] **SS4-3 非对称关系存在**：是否存在至少一个"敌人的敌人是朋友"或"貌合神离"的动态。完全对称的势力关系（A恨B，B恨A，C中立）过于简单。复杂的、非对称的关系网络更有叙事价值。

- [ ] **SS4-4 交叉依赖张力**：跨势力依赖（资源、知识、军事）是否制造了交叉张力。如果势力A控制粮食，势力B控制武器，势力C控制知识——三方互相需要又互相忌惮，这就是交叉依赖。交叉依赖让任何一方都无法单方面行动。

- [ ] **SS4-5 信息不对称识别**：势力间的信息不对称是否被识别。A知道B的秘密，B不知道A知道——这种信息差是政治剧情的核心燃料。每个势力应该有自己独有的信息优势和信息盲区。

- [ ] **SS4-6 关系演变路径**：势力关系是否有合理的演变路径（非静态）。当前的关系状态是结果，不是永恒——什么事件可能让盟友变成敌人？什么利益可能让敌人暂时合作？设定应该为关系演变留出空间。

### SS5: 势力动机与目标

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 每个势力为什么做它正在做的事？不是为了配合主角，而是为了自己。 |

**检查清单**：

- [ ] **SS5-1 核心利益明确**：每个主要势力是否有清晰的核心利益或目标。用一句话能说清"这个势力最想要什么"。如果说不清，说明势力设定还不够成熟。

- [ ] **SS5-2 目标内部一致**：势力目标是否内部一致。不能同时"追求和平"又"主动挑起战争"而没有解释。如果目标之间有张力，这个张力本身需要被设计（比如：短期目标与长期目标冲突）。

- [ ] **SS5-3 行为服务于自身利益**：势力在设定中的行为是否服务于自身目标（而非纯粹为了对抗主角）。反派势力不能只在主角出现时才行动——它应该有自己独立的行为逻辑和时间线。

- [ ] **SS5-4 隐藏议程存在**：至少一个势力有与其公开立场冲突的隐藏议程。公开说"保护人民"实际上在囤积资源；公开说"追求知识"实际上在制造信息不对称。隐藏议程在揭露时产生剧情冲击力。

- [ ] **SS5-5 目标间张力**：势力目标之间是否有张力（制造冲突潜力）。势力A想要X，势力B想要Y，但X和Y不可兼得——这就是目标间张力。目标间张力越大，势力冲突越激烈、越不可调和。

## III. 社会动态 (Social Dynamics)

> 类比：建筑的使用功能——人怎么在这个结构中生活、流动、互动。

### SS6: 阶层流动与固化

| 属性 | 值 |
|------|---|
| 严重级别 | warning |
| 核心问题 | 人能往上走吗？走不了的话，为什么不造反？ |

**检查清单**：

- [ ] **SS6-1 阶层定义清晰度**：社会阶层/等级是否清晰定义，而非简单的"富人和穷人"。至少应该有 3 个可区分的层级，每个层级有大致的人口比例、主要特征、核心诉求。

- [ ] **SS6-2 向上流动通道**：是否存在向上流动通道（即使被严重阻塞——设计必须明确）。完全封闭的阶层结构（零流动性）在现实中几乎不存在——总会有某种通道（军功、联姻、觉醒、宗教晋升）。关键不是有没有通道，而是通道有多窄、多不公平。

- [ ] **SS6-3 向下流动风险**：向下流动的风险是否被识别。什么会导致一个人从上层跌落？失去能力？政治斗争？经济破产？向下流动的风险影响上层的行为模式（焦虑、保守、排外）。

- [ ] **SS6-4 固化程度一致性**：阶层固化程度是否与权力结构和经济系统一致。如果经济系统高度垄断、权力结构高度集中，阶层应该高度固化。如果存在多种独立的经济来源和权力中心，阶层应该有更多流动性。两者必须匹配。

- [ ] **SS6-5 阶层认同影响**：阶层认同是否影响角色行为和跨阶层关系。不同阶层的人是否有不同的价值观、语言习惯、行为模式？跨阶层互动时是否有天然的隔阂或偏见？阶层认同的差异有丰富的叙事价值。

### SS7: 信息管控与知识不对称

| 属性 | 值 |
|------|---|
| 严重级别 | warning |
| 核心问题 | 谁知道什么？谁在阻止谁知道什么？ |

**检查清单**：

- [ ] **SS7-1 控制机制定义**：信息控制机制是否定义。审查制度、知识垄断（如文心派）、宗教教义、技术限制、地理隔离——至少有一种明确的机制。机制越具体，设定越可信。

- [ ] **SS7-2 知识不对称识别**：社会层级间的知识不对称是否被识别。统治者知道而底层不知道的？底层知道而统治者不知道的？专家群体独占的知识？知识不对称是权力不对称的基础。

- [ ] **SS7-3 信息泄露渠道**：是否存在信息泄露或非授权知识传播的渠道。没有任何信息控制系统是完美的——总会有裂缝：走私者、叛逃者、古书残页、口耳相传。这些渠道是主角获取关键信息的路径。

- [ ] **SS7-4 信息-冲突关联**：信息控制是否制造了至少一个剧情相关的冲突或张力。信息不对称导致误判、信息泄露导致信任危机、知识垄断导致阶层固化——信息问题应该与社会冲突有直接关联。

- [ ] **SS7-5 主角信息定位**：主角的信息优势/劣势是否相对于信息格局被定义。主角比其他人多知道什么？少知道什么？这种信息差如何影响他的行为和处境？主角的"现代知识碾压"或"穿越者信息差"应该被放在信息格局中考量。

### SS8: 主角嵌入点

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 主角在这个社会结构里站得住脚吗？有成长空间吗？ |

**检查清单**：

- [ ] **SS8-1 起始位置定义**：主角的起始社会位置是否清晰定义且有逻辑动机。不能是"主角恰好出现在这里"——他/她为什么在这个位置？是出生在这里、被流放到这里、还是主动选择？起始位置必须有因果关系。

- [ ] **SS8-2 多层接触能力**：起始位置是否提供接触至少 2 个不同社会层级或势力的通道。如果主角被锁死在一个层级里，剧情的空间就很小。起始位置应该像"十字路口"——主角能从多个方向接触社会结构的不同部分。

- [ ] **SS8-3 升级路径自然性**：社会结构是否为主角创造自然的升级路径。从底层到高层的每一步，是否都有社会结构内的逻辑支撑？（比如：拾荒者 → 通过试炼成为共鸣者 → 通过实力获得世家认可）。升级路径不能"跳跃"。

- [ ] **SS8-4 成长赋能一致性**：主角的成长轨迹是否被社会结构赋能（而非矛盾）。如果社会结构的设计是"底层永无翻身之日"，那主角的逆袭就需要有打破这个结构的具体条件（金手指、特殊身份、外部事件）。主角的成长不能与社会结构的设计矛盾。

- [ ] **SS8-5 独特张力利用**：是否存在至少一个主角 uniquely positioned 可利用或解决的社会张力。这个张力应该是：其他角色看不到/看不懂/无力解决，但主角因为其特殊身份、知识或能力可以介入。这是主角"不可替代性"的设定基础。

## 审计执行指南

### comprehensive 模式（首轮）

逐维度扫描全部 8 个维度，每个维度完成所有检查项。输出：
- 每维度 1-10 分评分 + 一句话评价
- 所有 critical/warning 问题列表（含位置、描述、修复方向）
- 亮点列表
- 完整审计报告 JSON

### focused 模式（回归审计）

1. 重点检查上一轮标记的维度（逐项验证修复情况）
2. 快速扫描其他维度是否有回归问题（新增问题或原有问题恶化）
3. 输出对比报告：
   - 每维度分数变化（↑↓→）
   - 上轮问题修复状态（已修复/部分修复/未修复/恶化）
   - 新发现问题列表
   - 趋势判断（整体改善/停滞/恶化）

### 评分标准（每维度 1-10）

| 分数 | 含义 |
|------|------|
| 9-10 | 出色，设计精巧且自洽，可作为同类设定的参考范例 |
| 7-8 | 良好，逻辑通顺，无明显漏洞，有少量可优化空间 |
| 5-6 | 及格，基本逻辑成立，但有明显短板需要修补 |
| 3-4 | 不足，存在逻辑硬伤或关键缺失，需要重点修复 |
| 1-2 | 严重缺失，该维度几乎未设计或完全不可信 |
