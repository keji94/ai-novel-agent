# 资源体系专项审计准出标准

---

## 收敛条件

```
converged = (
  critical_count == 0
  AND warning_count <= 2
  AND average_score >= 7.0
  AND min_dimension_score >= 4
)
```

与 social-structure-review 相同的收敛公式。四个条件必须同时满足。

## 评级标准

| 等级 | 总分 | Critical 数 | 最低维度分 | 含义 |
|------|------|-------------|-----------|------|
| **A** | ≥ 68 | 0 | ≥ 7 | 优秀。产消闭环、冲突有力、数据自洽 |
| **B** | ≥ 56 | 0 | ≥ 5 | 良好。大部分到位，小修即可 |
| **C** | ≥ 44 | ≤ 1 | ≥ 4 | 可接受。有明显短板，需修复 |
| **D** | < 44 或 critical ≥ 2 | — | — | 不足。需要重大修改或重新设计 |

> 总分 = 8 维度分数之和，满分 80。

## 循环参数

| 参数 | 值 | 说明 |
|------|---|------|
| 最大轮次 | 4 轮 | 与 social-structure-review 一致 |
| 首轮模式 | comprehensive | 全量扫描 8 维度 |
| 后续轮模式 | focused | 聚焦上轮问题 + 跨文件回归检查 |
| 修复 Agent | planner (mode: worldbuilding_fix) | 复用现有修复模式 |
| 修复优先级 | critical > warning > suggestion | |
| 跨文件联动 | 修复资源数据后同步检查社会结构、聚集地格局等文件 | RS8 专项检查 |

## 趋势追踪

与 social-structure-review 趋势追踪格式一致。

## 未收敛处理：用户检查点

4 轮循环后仍未收敛时，处理方式与 social-structure-review 一致。

## 与 social-structure-review 的区别

| 对比项 | 社会结构审计 | 资源体系审计 |
|--------|-------------|-------------|
| 维度数 | 8 (SS1-SS8) | 8 (RS1-RS8) |
| 满分 | 80 | 80 |
| 收敛条件 | 相同 | 相同 |
| 最大轮次 | 4 | 4 |
| 覆盖范围 | 社会结构文件 | 资源体系文件 |
| 跨文件检查 | 无（社会结构相对独立） | **有**（RS8 要求检查跨文件数值一致性） |
| 修复联动 | 修复社会结构不影响其他文件 | 修复资源数据可能需要同步更新社会结构、聚集地格局等文件 |

**重要**：资源体系的修复经常引发跨文件联动。当 planner 修改资源体系中的数据（如人口数字、资源占比）时，需要同步检查：
- `世界观设定-社会结构.md` 中的阶层人口数据
- `世界观设定-聚集地格局.md` 中的聚集地人口和资源描述
- `世界观设定-总览.md` 中的基本信息

focused 模式下，RS8 维度的检查范围自动扩展到这些关联文件。
