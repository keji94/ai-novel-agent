# 资源体系专项审计维度规范

本文件定义资源体系专项审计的 8 个维度、41 个检查项。独立于通用 15 维度体系，仅在资源体系专项审计时激活。

## I. 资源逻辑 (Resource Logic)

> 类比：建筑的管道系统——如果资源和消耗的管道对接不上，整个城市会"停水停电"。

### RS1: 资源稀缺性逻辑

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 资源为什么稀缺？稀缺程度是否与世界设定一致？ |

**检查清单**：

- [ ] **RS1-1 稀缺原因可解释**：每种核心资源的稀缺都有清晰的原因——是矿脉枯竭？封锁导致无法开采？加工能力丧失？还是消耗速度远超再生速度？不能"设定稀缺就稀缺"而不解释为什么。
- [ ] **RS1-2 稀缺程度与世界规则一致**：如果设定中存在强大的能力体系（如古物共鸣），资源稀缺不能与这种能力矛盾。例如：如果高阶共鸣者能"凭空创造物资"，那物资稀缺就不成立。稀缺必须是世界规则框架内的"无解问题"。
- [ ] **RS1-3 无"万能资源"漏洞**：是否存在一种资源可以解决所有问题？如果某种资源既能当武器、又能当食物、又能当货币、又能当能源——这不是资源设计，是作弊。每种核心资源应该有明确的局限性。
- [ ] **RS1-4 稀缺不可通过简单手段解决**：如果资源稀缺可以通过"多派几队人去采集"或"提高效率"来简单解决，那稀缺就不够"稀缺"。真正的稀缺应该是结构性瓶颈——无论怎么努力都无法显著增加供给。
- [ ] **RS1-5 替代品有限**：当一种资源枯竭时，是否有可用的替代品？如果每种资源都有完美的替代品，稀缺压力就被稀释了。好的设计是：关键资源没有好的替代品，迫使势力围绕它博弈。

### RS2: 产消平衡

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 生产和消耗是否平衡？有没有"凭空出现"或"去向不明"的物资？ |

**检查清单**：

- [ ] **RS2-1 生产渠道明确**：每种核心资源的获取方式是否具体定义（谁生产、用什么生产、在哪生产、产量大致多少）。不能只写"有农业""有采矿"，必须有可操作的生产链条。
- [ ] **RS2-2 消耗去向可追溯**：资源被消耗后的去向是否清楚。粮食被谁吃了？古物被谁用了？矿物被制成了什么？如果消耗端模糊，就无法判断供需是否平衡。
- [ ] **RS2-3 产消数据大致自洽**：生产总量和消耗总量之间是否大致平衡（允许缺口，缺口就是冲突来源）。如果生产远超消耗却没有解释"多出来的去哪了"，或者消耗远超生产却没有解释"差额怎么补的"，这是数据漏洞。
- [ ] **RS2-4 储存和流通有说明**：资源从生产到消费之间的储存和流通环节是否被描述。粮食怎么储存？能存多久？物资从产地到使用者的运输链是什么？储存损耗和运输损耗是否考虑？
- [ ] **RS2-5 关键瓶颈被识别**：产消链条中最脆弱的环节是否被明确指出。是生产不足？运输中断？储存损耗？还是消费端的需求增长太快？瓶颈就是冲突的焦点。
- [ ] **RS2-6 长期趋势不可逆（或可解释）**：资源衰减是单向的还是周期性的？如果是单向的，为什么不可逆？如果是周期性的，周期是什么？不可逆的衰减制造末世紧迫感，周期性的波动制造投机和囤积。

## II. 资源冲突 (Resource Conflict)

> 类比：建筑中的承重墙——资源冲突是剧情承重的结构墙，没有冲突就没有故事。

### RS3: 资源独特性

| 属性 | 值 |
|------|---|
| 严重级别 | warning |
| 核心问题 | 资源设计是否有辨识度？是否避免了"万能资源"和"换皮资源"？ |

**检查清单**：

- [ ] **RS3-1 核心资源不可互相替代**：排名前3的资源是否各自解决不同的问题。如果食物和能源和武器可以互相转换（比如"高级古物既能打人又能发光又能种地"），资源的区分度就不够。
- [ ] **RS3-2 至少一种资源有独特获取方式**：是否有一种资源的获取方式是这个世界独有的、不可在其他作品中找到的。独特获取方式是记忆点。
- [ ] **RS3-3 资源名称/概念有辨识度**：核心资源是否有好的命名和概念包装。读者能否用一句话记住"这个世界的资源体系是什么"。
- [ ] **RS3-4 资源体系服务于类型定位**：资源设计是否服务于目标类型。末世文的资源应该是"匮乏感驱动"，修仙文的资源应该是"境界突破驱动"。如果末世设定中资源过于充裕，就偏离了类型基调。
- [ ] **RS3-5 避免换皮陷阱**：资源体系是否只是"灵石→源晶""灵脉→矿脉"的命名替换，还是有本质差异。与同类作品（近3年热门）对比，是否有至少一个差异化特征。

### RS4: 资源驱动冲突

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 每种核心资源是否制造了势力间的冲突？还是只在数据表里存在？ |

**检查清单**：

- [ ] **RS4-1 每种核心资源有对应冲突**：排名前4的资源各自是否制造了至少一种势力间冲突。如果某种资源"很重要但没人抢"，要么它没有叙事价值，要么它的冲突设计被遗漏了。
- [ ] **RS4-2 冲突有具体的参与方**：每种资源冲突是否明确"谁和谁在争"。不能只写"资源稀缺导致冲突"，必须有具体的势力博弈——A势力控制产出、B势力控制加工、C势力控制流通。
- [ ] **RS4-3 冲突矩阵覆盖核心矛盾**：资源冲突矩阵是否覆盖了故事的核心矛盾（阶层冲突、内外冲突、势力博弈等）。如果核心矛盾在冲突矩阵中找不到对应的资源驱动，说明冲突矩阵不完整。
- [ ] **RS4-4 冲突有升级路径**：资源冲突是否有可能升级——从小摩擦到大冲突，从经济博弈到军事对抗。静态的冲突（永远同一烈度）缺乏叙事张力。
- [ ] **RS4-5 资源冲突与势力关系交叉**：资源冲突是否与社会结构中的势力关系网交叉。如果一个势力因为资源问题与另一个势力产生冲突，这个冲突是否叠加了已有的势力关系（如：公会与商会在盐务上的冲突，叠加了他们在物资调配上的依赖关系）。

### RS5: 资源层次与优先级

| 属性 | 值 |
|------|---|
| 严重级别 | warning |
| 核心问题 | 资源之间是否有明确的紧迫性排序？什么最先耗尽？什么最急？ |

**检查清单**：

- [ ] **RS5-1 资源有明确的优先级排序**：能否按"不立刻解决就会死人"的程度对资源排序。最紧迫的资源是什么？最不紧迫的？排序应该清晰且不矛盾。
- [ ] **RS5-2 不同势力有不同的资源优先级**：不同势力对资源的紧迫感是否不同。公会最缺什么？拾荒者最缺什么？商会最缺什么？如果所有势力都觉得同一种资源最紧迫，冲突就缺乏层次。
- [ ] **RS5-3 存在"木桶效应"资源**：是否有一种资源是"最短板"——不是最稀缺的，但它的短缺会导致其他所有资源的使用效率下降。比如：即使有粮食，没有盐也存不住；即使有武器，没有火药也用不了。
- [ ] **RS5-4 资源间的转换/依赖关系被定义**：资源之间是否有转换关系（A资源需要B资源才能加工成可用形态）或依赖关系（没有A资源就无法获取B资源）。这些关系创造了"一环扣一环"的连锁危机。
- [ ] **RS5-5 底层与高层的资源体验差异**：底层人和权贵对资源稀缺的体验是否不同。权贵可能只觉得古物稀缺，底层人可能同时面临缺粮、缺水、缺盐、缺药——这种体验差异本身就是阶层矛盾的放大器。

## III. 叙事适配 (Narrative Fit)

> 类比：建筑的功能设计——资源体系不仅要"站得住"，还要"好用"，为主角和剧情服务。

### RS6: 时间压力与紧迫感

| 属性 | 值 |
|------|---|
| 严重级别 | warning |
| 核心问题 | 资源衰减是否制造了叙事紧迫感？读者能感受到"时间在流逝"吗？ |

**检查清单**：

- [ ] **RS6-1 存在明确的倒计时**：资源体系是否提供了"如果不做X，Y就会在Z时间内发生"的倒计时。倒计时不需要精确数字（精确反而降低可信度），但趋势方向必须清晰。
- [ ] **RS6-2 时间预判有争议性**：不同势力对"还剩多少时间"是否有不同判断。如果只有一个"权威数字"，紧迫感会被稀释。争议性（有人说5年有人说50年）让主角和读者一起焦虑。
- [ ] **RS6-3 资源衰减有阶段性里程碑**：衰减不是匀速的，而是有加速点/拐点。比如"城墙力场衰退到某个阈值后将不可逆""盐源枯竭后食物保存能力归零"——这些里程碑是天然的剧情节点。
- [ ] **RS6-4 紧迫感在日常生活中可感知**：资源衰减是否体现在日常生活细节中（粮价上涨、配给缩减、灯火变暗），而不是只在数据表中存在。读者通过日常细节感受到紧迫，比通过数据报告更有效。
- [ ] **RS6-5 倒计时与主角节奏匹配**：资源倒计时的时间窗口是否与主角的成长节奏匹配。如果倒计时太长（50年），紧迫感不足；如果太短（1年），主角来不及成长。最佳状态是"刚好够，但容错率极低"。

### RS7: 主角资源交互

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 主角如何与资源体系交互？是资源体系的被动受害者还是能利用规则的破局者？ |

**检查清单**：

- [ ] **RS7-1 主角起始资源状态**：主角开局时处于资源链条的什么位置？是资源的生产者（拾荒者）、消费者（底层居民）、流通者（商人）、还是完全脱离资源体系的人？起始位置决定主角的第一个资源矛盾。
- [ ] **RS7-2 主角有独特的资源利用方式**：主角是否能以其他角色做不到的方式利用资源。比如：现代知识让主角识别被误分类的矿物、发现被忽略的资源替代品、或者发明更高效的使用方法。这种"独特利用"是金手指在资源层面的体现。
- [ ] **RS7-3 资源是主角成长的路径**：主角的资源获取是否与社会阶层提升同步。从底层拾荒者到体制内共鸣者，每一步是否都伴随着资源获取方式的升级？资源获取方式的升级应该体现主角社会地位的上升。
- [ ] **RS7-4 资源短缺制造主角的决策困境**：是否至少有一种资源短缺迫使主角在"道德"和"生存"之间做选择。比如：发现了一处盐矿，是自己用还是分给三环外的贫民？这种困境是角色深度的催化剂。
- [ ] **RS7-5 资源是剧情节点的触发器**：是否至少有3个剧情节点由资源事件直接触发（资源发现、资源争夺、资源枯竭、资源交易等）。如果资源体系只出现在背景设定中而不推动剧情，它就只是"装饰"。

### RS8: 跨文件数值一致性

| 属性 | 值 |
|------|---|
| 严重级别 | critical |
| 核心问题 | 资源数据在不同设定文件之间是否一致？同一个数字不能在A文件说50万、在B文件说100万。 |

**检查清单**：

- [ ] **RS8-1 人口数据一致**：人口数字在资源体系、社会结构、聚集地格局等文件中是否一致。如果社会结构说"三环外15-25万"、聚集地格局说"长安城80-100万"、资源体系说"总人口约100万"——需要确保加总对得上。
- [ ] **RS8-2 资源占比数据一致**：资源的来源占比（如食物来源占比）在不同文件中引用时是否保持一致。如果资源体系说"城内耕地40%"，社会结构中提到农业时不应该暗示不同的比例。
- [ ] **RS8-3 时间线一致**：资源衰减的时间节点与历史年表是否一致。如果年表说"纪元75年围困形成"，资源体系不应该说"纪元70年围困导致贸易中断"。
- [ ] **RS8-4 势力能力描述一致**：各势力的资源控制能力在不同文件中是否一致。如果资源体系说"匠造协会超然"、社会结构说"匠造协会依赖商会原材料"——这不矛盾，但需要确保理解上不冲突。
- [ ] **RS8-5 技术能力与资源消耗匹配**：如果设定中说某个势力拥有某种技术能力（如水力发电），资源体系中应该反映这种能力对资源的消耗。不能在技术文件里"有电"但在资源文件里"无能源消耗"。

## 审计执行指南

### comprehensive 模式（首轮）

逐维度扫描全部 8 个维度，每个维度完成所有检查项。输出：
- 每维度 1-10 分评分 + 一句话评价
- 所有 critical/warning 问题列表（含位置、描述、修复方向）
- 亮点列表
- 完整审计报告 JSON

### focused 模式（回归审计）

1. 重点检查上一轮标记的维度（逐项验证修复情况）
2. 快速扫描其他维度是否有回归问题
3. 特别注意 RS8（跨文件一致性）：修复一个文件中的资源数据后，检查其他文件是否需要同步更新
4. 输出对比报告

### 评分标准（每维度 1-10）

| 分数 | 含义 |
|------|------|
| 9-10 | 出色，资源体系精巧且自洽，可直接作为同类作品参考 |
| 7-8 | 良好，产消逻辑通顺，冲突驱动有力 |
| 5-6 | 及格，有基本框架但存在明显缺口 |
| 3-4 | 不足，关键资源数据缺失或产消逻辑断裂 |
| 1-2 | 严重缺失，资源体系几乎未设计或完全不可信 |
