# Reviser 记忆存储

本文件用于存储 Reviser (修订者) 的长期记忆。

## 常见修订模式

### OOC 修订
- 问题: 角色行为与人设不符
- 修复: 增加心理动机，或修改行为

### 设定冲突修订
- 问题: 与世界观设定矛盾
- 修复: 删除冲突内容，或调整设定

### AI 痕迹修订
- 问题: AI 套话、公式化结构
- 修复: 替换表达，打破公式化

## 修订经验

### 保留原意原则
- 修改后要保留原有的核心意图
- 不能因为修复问题而改变情节走向

### 最小修改原则
- 只修改有问题的部分
- 尽量不影响周边的正常内容

### 风格一致原则
- 修改后要保持原有的文风
- 不能因为修订而让风格突变

## AI 套话替换库

| AI 表达 | 人类表达 |
|---------|----------|
| 不得不承认 | 说实话、老实说 |
| 毋庸置疑 | 毫无疑问、不用问 |
| 显而易见 | 一眼看去、明眼人都知道 |
| 众所周知 | 大家都知道 |
| 首先...其次...最后 | （删除，直接叙述） |
| 全场震惊 | （具体描写） |
| 不禁 | （删除，直接描写） |
| 竟然 | （删除，或用其他表达） |
| 仿佛 | 像、如同（少用） |
| 宛如 | 像、似（少用） |

## 注意事项

- 修订后要验证 AI 痕迹是否增加
- 复杂问题可能需要多次修订
- rewrite 模式要谨慎使用
