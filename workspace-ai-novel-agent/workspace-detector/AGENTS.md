# Detector 协作流程定义

## 任务来源

| 来源 | 任务类型 | 输入内容 |
|------|----------|----------|
| Supervisor | 用户检测请求 | 章节范围 + 项目名 |
| Writer | 写后检测 | 章节内容 |
| Editor | 审计增强 | 章节内容 + 审计上下文 |

## 工作模式

### 模式 1: 单章检测

```
适用场景:
  - 检测单个章节的 AI 痕迹
  - 写后自动检测
  - Editor 审计辅助

输入:
  - 章节内容或章节号
  - 题材配置（疲劳词列表）

输出:
  - AI 痕迹得分 (0-100)
  - 问题定位
  - 修改建议

流程:
  1. 读取章节内容
  2. 执行 11 条确定性规则检测（零成本）
     ├── 禁止句式: 「不是...而是...」
     ├── 禁止破折号: 「——」
     ├── 转折词密度: 仿佛/忽然/竟然 等
     ├── 高疲劳词: 题材疲劳词超频
     ├── 元叙事: 编剧旁白式表述
     ├── 报告术语: 分析框架术语
     ├── 作者说教: 显然/不言而喻
     ├── 集体反应: 「全场震惊」套话
     ├── 连续了字: ≥6 句连续「了」
     ├── 段落过长: ≥2 段超 300 字
     └── 列表式结构: 首先/其次/最后
  3. 执行 4 项统计特征检测
     ├── TTR: 词汇多样性 (阈值 ≥ 0.05)
     ├── 句长标准差: 句式变化 (阈值 ≥ 10)
     ├── 段落长度标准差: 段落变化 (阈值 ≥ 30)
     └── 主动句比例: (阈值 ≥ 50%)
  4. 计算 AI 痕迹得分
  5. 生成检测报告
```

### 模式 2: 批量检测

```
适用场景:
  - 检测整本书的 AI 痕迹分布
  - 筛选高风险章节

输入:
  - 项目名
  - 章节范围: "all" 或 "1-50"

输出:
  - 整体 AI 痕迹平均分
  - 得分分布
  - 高风险章节列表
  - 按规则分类的问题统计

流程:
  1. 确定章节范围
  2. 逐章执行确定性规则检测
  3. 汇总统计
  4. 生成批量报告
```

### 模式 3: 增强检测

```
适用场景:
  - 确定性检测发现问题，需要更深入分析
  - 关键章节需要高质量检测
  - 修订前后对比

输入:
  - 章节内容
  - 题材配置
  - include_semantic: true

输出:
  - 确定性规则检测结果
  - 语义分析结果（5 个维度）
  - 统计特征分析
  - 综合 AI 痕迹得分

额外流程（在模式 1 基础上）:
  3.5. 执行语义分析（低成本 LLM）
       ├── 句式单调: 句式重复、变化少
       ├── 逻辑跳跃: 推理链断裂、突变
       ├── 情感平淡: 缺乏情感起伏
       ├── 描写空洞: 套话多、细节少
       └── 对话生硬: 不自然、信息堆砌
```

### 模式 4: 快速检测

```
适用场景:
  - Writer 写后自动检测
  - 只需快速判断是否需要深入检测

输入:
  - 章节内容

输出:
  - 通过/不通过
  - 仅 error 级别问题列表

流程:
  1. 仅运行 11 条确定性规则
  2. 不运行统计特征和语义分析
  3. 只报告 error 级别违规
  4. 返回简短结果
```

## 输出给其他 Agent

### 给 Supervisor

```json
{
  "status": "success",
  "summary": "第1章 AI痕迹得分 78/100（中等），2 个 error，3 个 warning",
  "suggestion": "建议修订"
}
```

```json
{
  "task": "根据检测结果修订",
  "chapter_number": 1,
  "ai_trace_score": 78,
  "priority_fixes": [

      "suggestion": "改写为直接叙述，去掉「不是...而是」结构"

  ]
}
```

```json
  "passed": false,

```

## 评分系统

```
AI 痕迹得分 = 100 - Σ(违规权重 × 次数)

得分解释:
- 90-100: 极低 AI 痕迹，通过
- 80-89: 低 AI 痕迹，通过
- 70-79: 中等 AI 痕迹，建议修订
- 60-69: 较高 AI 痕迹，需要修订
- <60: 高 AI 痕迹，建议重写
```

## 题材疲劳词配置

| 题材 | 疲劳词 |
|------|--------|
| 仙侠 | 冷笑、蝼蚁、倒吸凉气、瞳孔骤缩、天道、大道、因果、气运 |
| 玄幻 | 妖孽、逆天、震撼、恐怖、惊人、无法想象、不可思议 |
| 都市 | 恐怖如斯、不怒自威、气场、霸道 |

## 注意事项

### 检测精度

- 确定性规则准确率高，可直接作为修订依据
- 统计特征需结合上下文判断
- 语义分析为辅助参考，不可作为唯一依据

### 成本控制

- 快速检测/确定性检测：零 LLM 成本
- 语义分析检测：低成本（约 500 tokens/章）
- 批量检测建议先用确定性规则筛选，再对高风险章节做语义分析

### 批量检测策略

- 优先处理得分 < 70 的章节
- 同一规则频繁触发，说明是系统性问题，需全局调整
- 高风险章节建议使用 enhanced 模式复检
