# Operator 协作流程定义

## 任务来源

| 来源 | 任务类型 | 输入内容 |
|------|----------|----------|
| Supervisor | 运营咨询 | 咨询问题 |
| Planner | 市场参考 | 类型/风格 |
| Analyst | 数据支持 | 分析需求 |

## 分析模式

### 模式1: 市场趋势分析

```
输入: 分析维度(类型/平台/时期)
输出: 市场分析报告
流程:
  1. 收集市场数据
  2. 分析趋势变化
  3. 识别机会风险
  4. 给出建议
```

### 模式2: 读者研究

```
输入: 目标类型/作品
输出: 读者研究报告
流程:
  1. 收集读者数据
  2. 分析偏好行为
  3. 识别雷区爽点
  4. 给出建议
```

### 模式3: 运营策略

```
输入: 作品情况/目标
输出: 运营策略建议
流程:
  1. 分析现状
  2. 评估资源
  3. 制定策略
  4. 执行建议
```

## 协作接口

### 接收任务

```json
{
  "task": "市场分析",
  "scope": {
    "genre": "仙侠",
    "platform": "起点",
    "time_range": "近3个月"
  },
  "focus": ["趋势", "竞争", "机会"]
}
```

### 输出结果

```json
{
  "status": "completed",
  "report": {
    "market_overview": {...},
    "trends": [...],
    "competition": {...},
    "opportunities": [...],
    "risks": [...]
  },
  "recommendations": [
    {
      "action": "建议动作",
      "reason": "原因",
      "priority": "high/medium/low"
    }
  ]
}
```

## 分析维度

### 市场维度

| 维度 | 内容 | 数据来源 |
|------|------|----------|
| 类型热度 | 各类型占比变化 | 平台榜单 |
| 新书表现 | 新书数据分布 | 新书榜 |
| 完本表现 | 完本作品影响 | 完本榜 |
| 作者分布 | 头部/腰部作者情况 | 作者榜 |

### 读者维度

| 维度 | 内容 | 数据来源 |
|------|------|----------|
| 画像 | 年龄、性别、地域 | 平台数据 |
| 偏好 | 类型、风格、节奏 | 阅读数据 |
| 行为 | 时段、频率、付费 | 行为数据 |
| 反馈 | 评论、打赏、推荐 | 互动数据 |

### 运营维度

| 维度 | 内容 | 数据来源 |
|------|------|----------|
| 更新 | 频率、时段、字数 | 作品数据 |
| 推荐 | 类型、效果、方式 | 推荐位 |
| 互动 | 评论、粉丝、社群 | 社交数据 |
| 变现 | 订阅、打赏、版权 | 收入数据 |
