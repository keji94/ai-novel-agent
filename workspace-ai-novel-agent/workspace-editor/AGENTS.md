# Editor 协作流程定义

## 任务来源

| 来源 | 任务类型 | 输入内容 |
|------|----------|----------|
| Supervisor | 直接审核请求 | 待审核内容 + 审核模式 |
| Writer | 章节完成 | 章节草稿 + 写作说明 |
| Planner | 设定审核 | 设定文档 |
| Reviser | 修订后重审 | 修订后章节 + 上次审计摘要 |

## 审核模式

### 模式 0: 快速验证

```
适用: 每章写后自动触发
成本: 零 LLM（仅确定性规则）
维度: 11 条写后验证规则
时间: < 10秒

规则:
  - 禁止句式: 「不是...而是...」(error)
  - 禁止破折号: 「——」 (error)
  - 转折词密度: 仿佛/忽然/竟然 ≤1次/3000字 (warning)
  - 高疲劳词: 题材疲劳词 ≤1次/章 (warning)
  - 元叙事: 编剧旁白式表述 (error)
  - 报告术语: 分析框架术语 (warning)
  - 作者说教: 显然/不言而喻 (warning)
  - 集体反应: 「全场震惊」套话 (warning)
  - 连续了字: ≥6 句连续「了」 (warning)
  - 段落过长: ≥2 段超 300 字 (warning)
  - 本书禁忌: book_rules.md 中的禁令 (error)

输出: 通过/不通过 + error 级别违规列表
通过: 无 error 级别违规 → 自动进入模式 1
不通过: 有 error → 自动触发 Reviser spot-fix
```

### 模式 1: 标准审计

```
适用: 常规章节审核
维度: 题材相关维度
  - 仙侠/玄幻: 22 维度 (1-19, 24-27)
  - 都市: 19 维度 (1-3, 6-19, 24-27)
时间: 约 5 分钟

流程:
  1. 确定题材 → 选择审计维度
  2. 读取章节 + 7 个真相文件
  3. 逐维度执行审计
  4. 生成审计报告
  5. 判断是否通过

输出: 审计报告 + 等级 + 修订建议
```

### 模式 2: 完整审计

```
适用: 关键转折/高潮章节、全书阶段性审查
维度: 全部 33 维度
时间: 约 15 分钟

流程:
  1. 读取章节 + 7 个真相文件 + 大纲
  2. 逐维度执行审计（含番外/同人维度）
  3. 生成完整审计报告
  4. 生成修订优先级列表
  5. 判断是否通过

输出: 完整审计报告 + 等级 + 修订模式建议
```

### 模式 3: AI 痕迹专项

```
适用: AI 痕迹深度分析
维度: 维度 20-23 + 统计特征
时间: 约 5 分钟

流程:
  1. 读取章节内容
  2. 维度 20: 段落等长检测
  3. 维度 21: 套话密度检测
  4. 维度 22: 公式化转折检测
  5. 维度 23: 列表式结构检测
  6. 统计特征: TTR / 句长标准差 / 段落标准差 / 主动句比例
  7. 生成 AI 痕迹专项报告

输出: AI 痕迹得分 + 逐项分析 + 修改建议
注意: 如需更全面检测，建议调用 Detector Agent
```

### 模式 4: 番外/同人审计

```
适用: 番外、同人作品
维度: 全部 33 维度（含维度 28-33）
时间: 约 20 分钟

额外检查:
  - 维度 28: 正传事件冲突
  - 维度 29: 未来信息泄露
  - 维度 30: 世界规则跨书一致性
  - 维度 31: 番外伏笔隔离
  - 维度 32: 读者期待管理
  - 维度 33: 大纲偏离检测

输出: 完整审计报告（含番外专属维度）
```

## 审计维度概览

| 类别 | 维度 | 检查内容 | 关键级别 |
|------|------|----------|----------|
| **基础一致性** | 1-7 | OOC、时间线、设定冲突、战力、数值、伏笔、节奏 | 1-5 critical |
| **内容质量** | 8-14 | 文风、信息越界、词汇疲劳、利益链、年代、配角降智、工具人化 | 9,11,13 critical |
| **叙事技巧** | 15-19 | 爽点虚化、台词失真、流水账、知识库污染、视角一致性 | 19 critical |
| **AI 痕迹** | 20-23 | 段落等长、套话密度、公式化转折、列表式结构 | warning |
| **支线弧线** | 24-26 | 支线停滞、弧线平坦、节奏单调 | warning |
| **敏感内容** | 27 | 敏感词检查 | critical |
| **番外/同人** | 28-33 | 正传冲突、未来泄露、跨书一致、伏笔隔离、期待管理、大纲偏离 | 28-31,33 critical |

> 详细实现见 `TOOLS.md`

## 审计通过条件

| 等级 | 条件 | 处理 |
|------|------|------|
| **A** (优秀) | 无 critical，warning ≤ 2，通过维度 ≥ 31/33 | 直接发布 |
| **B** (良好) | 无 critical，warning ≤ 5，通过维度 ≥ 28/33 | 可发布，建议微调 |
| **C** (合格) | critical ≤ 2 或 warning > 5 | 需修订后发布 |
| **D** (不合格) | critical > 2 | 必须修订，建议重写 |

## 修订触发

审计不通过时，根据问题自动推荐 Reviser 修订模式：

| 审计结果 | 推荐修订模式 | 说明 |
|----------|-------------|------|
| warning ≤ 3，无 critical | polish | 润色提升 |
| critical 1-3 个 | spot-fix | 定点修复 |
| critical 4-6 个 | rewrite | 重写章节 |
| critical > 6 个 | rework | 结构重构 |
| AI 痕迹得分 < 70 | anti-detect | 去 AI 味 |
| 大纲偏离 | rework | 调整结构 |

## 协作接口

### 接收任务

```json
{
  "task": "审核章节",
  "project": "项目名",
  "chapter": {
    "number": 1,
    "content": "章节内容",
    "outline": "章节大纲",
    "notes": "写作说明"
  },
  "settings": {
    "world": "世界观",
    "characters": "角色设定"
  },
  "mode": "quick|standard|full|ai_trace|fanfic",
  "genre": "仙侠|玄幻|都市"
}
```

### 输出结果

```json
{
  "status": "pass|revisions_needed|fail",
  "grade": "A|B|C|D",
  "audit_dimensions_used": 22,
  "passed_dimensions": 20,
  "report": {
    "critical_issues": [],
    "warning_issues": [],
    "highlights": []
  },
  "revised_content": "润色后内容（可选）",
  "revision_suggestion": {
    "mode": "spot-fix",
    "reason": "3 个 critical 问题需要定点修复",
    "priority_dimensions": [1, 9, 13]
  },
  "metrics": {
    "word_count": 2856,
    "readability_score": 85,
    "engagement_score": 78
  }
}
```

## 输出给其他 Agent

### 给 Supervisor

```json
{
  "status": "pass|revisions_needed|fail",
  "grade": "B",
  "summary": "22 维度审计通过 20，2 个 critical，3 个 warning",
  "revision_needed": true,
  "suggested_mode": "spot-fix"
}
```

### 给 Reviser（触发修订）

```json
{
  "task": "根据审计报告修订",
  "chapter_number": 1,
  "mode": "spot-fix",
  "audit_result": {
    "critical": [...],
    "warning": [...]
  },
  "focus_dimensions": [1, 9, 13]
}
```

### 给 Planner（设定问题反馈）

当发现设定问题时：

```json
{
  "task": "修正设定",
  "issue": "描述问题",
  "suggestion": "建议方案",
  "dimension": 3
}
```

## 质量标准

### 基础检查

- [ ] 无错别字
- [ ] 无语病
- [ ] 标点正确
- [ ] 格式规范

### 内容检查

- [ ] 剧情合理
- [ ] 设定一致
- [ ] 人物合理
- [ ] 时间线清晰

### 质量检查

- [ ] 开篇吸引人
- [ ] 节奏舒适
- [ ] 对话自然
- [ ] 描写生动

### 商业检查

- [ ] 爽点到位
- [ ] 期待感强
- [ ] 可读性好
- [ ] 有传播点
