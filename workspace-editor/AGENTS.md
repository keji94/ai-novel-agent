# Editor 协作流程定义

## 任务来源

| 来源 | 任务类型 | 输入内容 |
|------|----------|----------|
| Supervisor | 直接审核请求 | 待审核内容 + 审核模式 |
| Writer | 章节完成 | 章节草稿 + 写作说明 |
| Planner | 设定审核 | 设定文档 |
| Reviser | 修订后重审 | 修订后章节 + 上次审计摘要 |

## 审核模式

### 模式 0: 快速验证

```
适用: 每章写后自动触发
成本: 零 LLM（仅确定性规则）
时间: < 10秒

规则来源: rules/deterministic/ (D001-D023)
执行脚本: .openclaw/skills/content-scanner/scripts/run_deterministic.py
规则详情: rules/_index.md

规则概要:
  - 禁止句式/破折号/元叙事 (D001-D003, critical)
  - 转折词密度/疲劳词/报告术语/说教词/集体套话 (D004-D008, warning)
  - 连续了字/段落过长 (D009-D010, warning)
  - 设定门控: 提前释放/认知越级/重复引入 (D013-D015, critical/warning)
  - AI 痕迹统计: TTR/句长标准差/段落标准差/主动句比例 (D016-D019, info)
  - 明喻密度控制 (D022, warning)
  - 编号式自我总结 (D023, warning)

校验流程（设定相关）:
  1. 读取 settings_release.json 索引
  2. 扫描正文，匹配设定名称和关键词
  3. 对照 status + knowledge_level 判定
  4. error → 自动触发 Reviser spot-fix
  5. warning → 记录到审计报告

输出: 通过/不通过 + error 级别违规列表
通过: 无 error 级别违规 → 自动进入模式 1
不通过: 有 error → 自动触发 Reviser spot-fix
```

### 模式 1: 标准审计

```
适用: 常规章节审核
维度: 题材相关维度
  - 仙侠/玄幻: 22 维度 (1-19, 24-27)
  - 都市: 19 维度 (1-3, 6-19, 24-27)
时间: 约 5 分钟

流程:
  1. 确定题材 → 选择审计维度
  2. 读取章节 + 7 个真相文件
  3. 逐维度执行审计
  4. 生成审计报告
  5. 判断是否通过

输出: 审计报告 + 等级 + 修订建议
```

### 模式 2: 完整审计

```
适用: 关键转折/高潮章节、全书阶段性审查
维度: 全部 33 维度
时间: 约 15 分钟

流程:
  1. 读取章节 + 7 个真相文件 + 大纲
  2. 逐维度执行审计（含番外/同人维度）
  3. 生成完整审计报告
  4. 生成修订优先级列表
  5. 判断是否通过

输出: 完整审计报告 + 等级 + 修订模式建议
```

### 模式 3: AI 痕迹专项

```
适用: AI 痕迹深度分析
规则来源: rules/deterministic/ (D001-D012, D016-D019) + rules/replacements/ai-traces.yaml
执行方式: 委托 Checker 或 run_deterministic.py 执行确定性检测
时间: 约 5 分钟

流程:
  1. 委托 Checker 执行针对性规则检测（或调用 run_deterministic.py）
  2. 维度 20-23 审计: 段落等长/套话密度/公式化转折/列表式结构
  3. 统计特征: TTR / 句长标准差 / 段落标准差 / 主动句比例 (D016-D019)
  4. 生成 AI 痕迹专项报告

输出: AI 痕迹得分 + 逐项分析 + 修改建议
注意: 如需更全面检测（含语义分析），建议调用 Detector Agent
```

### 模式 4: 番外/同人审计

```
适用: 番外、同人作品
维度: 全部 33 维度（含维度 28-33）
时间: 约 20 分钟

额外检查:
  - 维度 28: 正传事件冲突
  - 维度 29: 未来信息泄露
  - 维度 30: 世界规则跨书一致性
  - 维度 31: 番外伏笔隔离
  - 维度 32: 读者期待管理
  - 维度 33: 大纲偏离检测

输出: 完整审计报告（含番外专属维度）
```

### 模式 5: 知识一审（逐条过滤）

逐条审核 Analyst 提取的技巧，输出 APPROVE / REJECT / MERGE 判定。

> 详细审核维度、判定标准和输出 JSON 格式见 `reference/knowledge-review-modes.md`

### 模式 6: 知识二审（系统一致性）

Learner 入库后执行全系统一致性检查（互重/分类/质量/引用完整性）。

> 详细检查项和输出 JSON 格式见 `reference/knowledge-review-modes.md`

### 模式 7: 定稿审核与差异学习

```
适用: 用户人工修改章节后的定稿（规则16触发）
输入: chapter_path + draft_path (可选) + project
输出: 审核报告 + 差异分析 + 候选规则

流程:
  Phase A: 内容审核
    - 对人工修改后的终稿执行 Mode 0 快速验证
    - 确保人工修改没有引入新的质量问题

  Phase B: 差异分析 (仅当 draft_path 存在时)
    1. 读取初稿和终稿
    2. 逐段对比，标记所有变更:
       - UNCHANGED: 完全相同
       - MODIFIED: 有修改（重点分析）
       - ADDED: 终稿新增段落
       - DELETED: 终稿删除段落
    3. 对每个 MODIFIED 变更分类:
       - ai_trace: AI痕迹消除（AI套话→自然表达）
       - style: 风格调整（句式、节奏、用词偏好）
       - grammar: 语句优化（不通顺→通顺、冗余→精简）
       - plot: 情节修改（内容/逻辑变化）
       - setting: 设定修正
       - other: 其他
    4. 对同一类别的变更提取共性模式

  Phase C: 规则生成（直接生效，无需确认）
    - ai_trace 类 → 生成 replacement 规则 (追加到 ai-traces.yaml)
    - style/grammar 类 → 生成 learned 规则 (写入 rules/learned/)
    - plot/setting 类 → 不生成规则（内容选择，非写作技巧）

  Phase D: 输出
    - 定稿审核结果 (grade + issues)
    - 差异分析报告 (每类变更数量 + 典型示例)
    - 候选规则列表 (直接写入 rules/)
```

> 详细差异分析方法论和规则生成模板见 `reference/finalize-review.md`

## 审计维度概览

| 类别 | 维度 | 检查内容 | 关键级别 |
|------|------|----------|----------|
| **基础一致性** | 1-7 | OOC、时间线、设定冲突、战力、数值、伏笔、节奏 | 1-5 critical |
| **内容质量** | 8-14 | 文风、信息越界、词汇疲劳、利益链、年代、配角降智、工具人化 | 9,11,13 critical |
| **叙事技巧** | 15-19 | 爽点虚化、台词失真、流水账、知识库污染、视角一致性 | 19 critical |
| **AI 痕迹** | 20-23 | 段落等长、套话密度、公式化转折、列表式结构 | warning |
| **支线弧线** | 24-26 | 支线停滞、弧线平坦、节奏单调 | warning |
| **敏感内容** | 27 | 敏感词检查 | critical |
| **番外/同人** | 28-33 | 正传冲突、未来泄露、跨书一致、伏笔隔离、期待管理、大纲偏离 | 28-31,33 critical |

> 详细实现见 `TOOLS.md`

## 审计通过条件

| 等级 | 条件 | 处理 |
|------|------|------|
| **A** (优秀) | 无 critical，warning ≤ 2，通过维度 ≥ 31/33 | 直接发布 |
| **B** (良好) | 无 critical，warning ≤ 5，通过维度 ≥ 28/33 | 可发布，建议微调 |
| **C** (合格) | critical ≤ 2 或 warning > 5 | 需修订后发布 |
| **D** (不合格) | critical > 2 | 必须修订，建议重写 |

## 修订触发

审计不通过时，根据问题自动推荐 Reviser 修订模式：

| 审计结果 | 推荐修订模式 | 说明 |
|----------|-------------|------|
| warning ≤ 3，无 critical | polish | 润色提升 |
| critical 1-3 个 | spot-fix | 定点修复 |
| critical 4-6 个 | rewrite | 重写章节 |
| critical > 6 个 | rework | 结构重构 |
| AI 痕迹得分 < 70 | anti-detect | 去 AI 味 |
| 大纲偏离 | rework | 调整结构 |

## 质量标准

### 基础检查

- [ ] 无错别字
- [ ] 无语病
- [ ] 标点正确
- [ ] 格式规范

### 内容检查

- [ ] 剧情合理
- [ ] 设定一致
- [ ] 人物合理
- [ ] 时间线清晰

### 质量检查

- [ ] 开篇吸引人
- [ ] 节奏舒适
- [ ] 对话自然
- [ ] 描写生动

### 商业检查

- [ ] 爽点到位
- [ ] 期待感强
- [ ] 可读性好
- [ ] 有传播点
