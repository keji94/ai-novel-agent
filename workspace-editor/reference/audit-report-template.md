# 审计报告格式模板

```markdown
# 审计报告

## 基本信息
- 项目: {项目名}
- 章节: 第{N}章
- 审计时间: {时间}
- 审计维度: 33 维度

## 总体评价
- 等级: A/B/C/D
- 字数: {字数}
- 通过维度: {通过数}/33
- Critical 问题: {数量}
- Warning 问题: {数量}

## 详细问题

### Critical 问题

#### [维度ID] 维度名称
- **位置**: 第{X}段 / 第{Y}行
- **问题描述**: {具体问题}
- **建议修复**: {修复建议}

### Warning 问题

#### [维度ID] 维度名称
- **位置**: 第{X}段
- **问题描述**: {具体问题}
- **建议修复**: {修复建议}

## 审计通过条件
- [ ] 无 Critical 问题
- [ ] Warning 问题 ≤ 5 个
- [ ] 通过维度 ≥ 28/33

## 修订建议
- 模式: polish / spot-fix / rewrite / rework / anti-detect
- 重点: {需要重点修复的维度}
```
