# 写作方法论总览

## 一、起承转合四幕结构

起承转合是中国叙事的核心框架，在爽文中需要在三个层级上运用。

### 单元故事层（1-2万字）

最小叙事单元，解决一个核心冲突。

| 阶段 | 功能 | 章数占比 | 关键动作 |
|------|------|---------|---------|
| **起** | 建立目标 | 10-20% | 主角要什么？什么阻碍他？为什么重要？ |
| **承** | 全面铺设 | 40-50% | 角色登场、动机建立、情绪亏欠、信息差制造 |
| **转** | 逆转/揭示 | 15-20% | 信息揭示、力量展示、格局变化 |
| **合** | 释放+钩子 | 20-25% | 爽点释放、延续爽感、设置下一单元钩子 |

### 卷层（8-15万字）

包含多个单元故事，有独立的起承转合弧线：

- **卷起**（3-5章）：建立卷级目标，引入新环境/新角色
- **卷承**（主体）：多个单元故事串联，逐步升温
- **卷转**（5-10章）：卷级大逆转，格局剧变
- **卷合**（3-5章）：卷级爽点释放 + 收束伏笔 + 下一卷钩子

### 全书层（300万字+）

- **全书起**（第一卷）：世界观建立 + 金手指激活 + 读者期待建立
- **全书承**（中间卷）：格局扩展 + 角色深化 + 多线交织
- **全书转**（倒数第二卷）：终极真相揭示 + 道德两难
- **全书合**（最后一卷）：终极抉择 + 主题升华 + 结局

---

## 二、多层级结构框架

```
全书（300万字）
└── 卷（40-55万字 × N）
    └── 单元故事（8-15万字 × 3-6）
        └── 章（2000-3000字 × 30-50）
```

### 层级间的衔接规则

1. **单元故事→卷**：每个单元故事服务于卷级目标，同时有自己的完整性
2. **卷→全书**：每卷推进全书主线，同时卷内弧线完整可读
3. **信息递进**：读者在每个层级都能获得"这一层级的满足感"，同时被下一层级的悬念牵引

---

## 三、换图系统（长篇可持续性）

300万字长篇的核心挑战是"如何让故事不重复"。换图系统是网文界的成熟方案。

### 原理

设计自包含的"地图"或"领域"，每个地图有：
- 完整弧线（8-15万字）
- 本地权力结构
- 本地冲突
- 成长机会
- 与主线的连接点

### 换图触发条件

- 当前地图的冲突已解决
- 主角成长需要新环境
- 世界观需要扩展
- 读者审美疲劳（通常5-8个单元故事后）

### 换图注意事项

- 新地图不能是旧地图的简单复制（不同权力结构、不同冲突类型）
- 旧地图的角色可以有选择地跟随（不是全部重置）
- 主线伏笔必须跨地图延续

## 四、力量体系设计原则

### 核心要求

1. **清晰层级**：每个等级有明确的能力边界
2. **质变而非量变**：等级之间是本质变化，不只是数值增加
3. **进阶有代价**：不是自动升级，需要条件/机缘/代价
4. **跨级可能但有条件**：弱胜强需要策略/金手指/牺牲，不能无代价
5. **体系一致性**：不出现前后矛盾的设定

### 模板

```
一阶 [名称]：基础能力，能做什么，不能做什么
二阶 [名称]：新能力类型出现（质变点）
三阶 [名称]：能力质变，应用范围扩大
...
每个等级：
- 进入条件（需要什么）
- 新增能力（获得什么）
- 战力范围（大概水平）
- 弱点/限制（不能做什么）
```

### 长篇注意事项

- **爽点通胀应对**：高等级不能只是"更大的数字"，要有新的玩法维度
- **力量天花板**：设定明确的已知极限，避免无限膨胀
- **非战斗维度**：力量体系不只是战斗力，还有感知、创造、社交等维度

## 五、伏笔管理

### 三级分层

| 层级 | 跨度 | 数量建议 | 管理 |
|------|------|---------|------|
| 长线 | 贯穿全书 | 5-8条 | 需要追踪表，明确埋设和收束节点 |
| 中线 | 单卷内 | 每卷3-5条 | 卷大纲中标注埋设和揭晓位置 |
| 短线 | 几章内 | 不限 | 写作时灵活处理 |

### 伏笔设计原则

1. **有意义的伏笔**：收束时必须推动剧情或深化角色，不能只是"炫技"
2. **可追溯性**：读者回头看时能找到埋设点（不需要重读全书，但能意识到"原来如此"）
3. **多重解读**：好的伏笔在揭晓前可以有多种解读方向
4. **不无限拖延**：长线伏笔也必须有大致的揭晓时间窗口

### 伏笔陷阱

- **埋了忘收**：必须有追踪表
- **收束太随意**：揭晓必须有冲击力，不能草草了事
- **伏笔过密**：读者记不住，失去意义
- **为伏笔而伏笔**：伏笔应该自然融入剧情，不是硬塞

## 六、长篇陷阱应对

### 1. 爽点通胀

**问题**：读者对爽点阈值越来越高，同样的套路越来越不管用。

**应对**：
- 变换爽点类型（不总是同一种打脸模式）
- 引入情感类爽点（感动、遗憾、释然）调节节奏
- 增加智力型爽点（推理、策略、布局）
- 升级赌注（从个人到群体到文明）

### 2. 力量膨胀

**问题**：主角越来越强，战斗失去张力。

**应对**：
- 设定明确的力量天花板
- 引入新的力量维度（不只是战斗力）
- 聚焦"如何运用力量"而非"力量有多大"
- 用非战斗冲突替代纯战斗（政治、道德、社交）

### 3. 世界观膨胀

**问题**：设定越来越多，读者记不住，作者管不了。

**应对**：
- 新设定必须与剧情冲突直接挂钩（不能只是"设定集"）
- 旧设定在新场景中复用（换皮不换骨）
- 控制活跃设定数量（同时活跃的设定系统 ≤ 3个）
- 定期收束已展开的设定线

### 4. 角色失控

**问题**：配角越来越多，有的变成工具人，有的抢主角风头。

**应对**：
- 控制核心角色数量（有名字有戏份的活跃角色 ≤ 20-30）
- 配角有退场机制（完成弧线后离开，不是永远跟着）
- 配角有独立动机（不是围着主角转）
- 定期清理不活跃角色

### 5. 剧情重复

**问题**：换地图后故事模式重复（"新的地方→被看不起→打脸→离开"）。

**应对**：
- 每个新地图有**不同的冲突类型**（见 conflict-design.md）
- 主角在新地图中的角色不同（从挑战者变为被挑战者、从局外人变为局中人等）
- 旧地图的遗留问题在新地图中发酵（不完全重置）
