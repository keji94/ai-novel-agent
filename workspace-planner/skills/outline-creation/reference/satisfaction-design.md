# 爽点设计手册

## 一、爽点定义

爽点 = 任何让读者获得情感满足的事件。不限于"打脸"，包括感动、搞笑、解谜、浪漫等多种情绪。

---

## 二、13 类爽点详解

### 类型 A：剧情高峰
**核心**：超出预期的剧情发展
- 意想不到的剧情反转
- 高强度动作场景
- 情感困境的解决
- **示例**：主角被逼入绝境时，一个意想不到的盟友出现

### 类型 B：高调碾压
**核心**：绝对实力/能力的正面展示
- 挑战极限并成功
- 瓶颈突破
- 完美化解危机
- 压制性击败敌人
- 领导力展示
- **示例**：主角在考核中用远超标准的成绩碾压所有对手

### 类型 C：低调深沉
**核心**：读者知道主角的底牌，其他角色不知道
- 幕后操控
- 暗中布局
- 关键时刻突然出手
- **示例**：所有人都以为主角是废物，但读者知道他一直在暗中准备

### 类型 D：扮猪吃虎
**核心**：故意隐藏实力→被轻视→展示实力→打脸
- 装弱被欺负
- 忍辱负重
- 关键时刻揭示实力
- **示例**：主角在拍卖会上被富二代嘲笑买不起，然后掏出压轴拍品

### 类型 E：意外惊喜
**核心**：天降好事
- 突然获得强力盟友
- 意外获得宝物/能力
- 从天而降的好运
- **注意**：不能过度使用，否则失去意义。建议与铺垫结合（"惊喜"有"伏笔"支撑）

### 类型 F：突破常规
**核心**：打破读者预期
- 类型混搭元素
- 严肃场景中的幽默
- 颠覆读者对角色/事件的认知
- **示例**：大决战前主角和反派先坐下来喝杯茶

### 类型 G：脱离掌控
**核心**：从困境中挣脱
- 摆脱反派控制
- 发现反派弱点
- 角色成长/觉醒
- 战斗中逆转
- 陷阱逃脱
- **示例**：主角被囚禁后，用之前隐藏的能力打破牢笼

### 类型 H：救世主
**核心**：大规模的拯救/改变
- 拯救世界/城市/群体
- 两个对立势力的调解
- 大规模挑战的成功
- **示例**：主角带领联盟军击退异族大军

### 类型 I：反套路
**核心**：完全出人意料的发展
- 复杂多层角色
- 非线性叙事
- 需要深度思考的隐藏线索
- **注意**：适度使用，过度会让读者疲惫

### 类型 J：推理叠加
**核心**：信息逐步拼合的智力快感
- 逐步解开谜题
- 不断添加新线索
- 世界观/维度变化
- 想象空间的扩展
- **示例**：主角通过多个线索推断出幕后黑手的身份

### 类型 K：掐高潮
**核心**：故意延后爽点释放以增强张力
- 压制高潮建立张力
- 跳切叙事
- 情感前置
- 延长铺垫
- 角色性格转变的铺垫
- **示例**：主角即将突破但被打断，读者急得不行，三章后终于突破

### 类型 L：揭秘
**核心**：真相揭示的震撼感
- 逐步揭示真相
- 突然反转
- 对峙中揭示信息
- 时间压力下的揭秘
- **示例**：主角发现自己的身世远比想象中复杂

### 类型 M：配角魅力
**核心**：配角的个人魅力时刻
- 逐步展现个性
- 极端特质放大
- 与主角的对比
- 幽默互动
- 动态关系变化
- **示例**：铁牛在严肃场合说出一句让人喷饭的话

## 三、爽点设计公式

### 核心公式

```
铺垫 → 转折 → 爽点 → 延续爽点
```

### 3.1 铺垫（制造情绪亏欠）

铺垫的本质是让读者**想要**满足感。四种手段：

| 手段 | 说明 | 示例 |
|------|------|------|
| **信息差** | 读者知道但角色不知道（或反过来） | 读者知道主角隐藏了实力，看着反派嚣张 |
| **权力差** | 强弱悬殊，弱者被压迫 | 独臂少年被一群壮汉围住 |
| **时间压力** | 倒计时，不行动就会失去 | "三天后就是公会考核，你还差一个条件" |
| **赌注升级** | 失败的代价越来越大 | 从"丢了面子"升级到"丢了性命"再到"丢了整个城市" |

**关键原则**：铺垫要"快、准、狠"——不是越长越好，而是要精准地制造情绪亏欠。

### 3.2 转折（信息/格局变化）

转折是铺垫和爽点之间的桥梁：
- 揭示隐藏信息（"你以为我只是个废物？"）
- 亮出底牌（主角展示金手指）
- 颠覆格局（局势突然变化）
- 外力介入（意想不到的盟友/事件）

### 3.3 爽点（情绪释放）

满足铺垫制造的亏欠：
- 释放积累的张力
- 实现铺垫的承诺
- 交付期待中的满足

### 3.4 延续爽点（余韵+钩子）

爽点释放后不能立刻结束，要有余韵：
- **数收获**：盘点获得的战利品/能力/声望
- **发现额外奖励**：意料之外的好处
- **旁观者震惊**：其他角色的反应（"这……这不可能！"）
- **连锁反应**：爽点的后果扩散
- **下一阶段钩子**：暗示更大的挑战即将到来

## 四、爽点密度规则

### 按平台区分

| 平台 | 大爽点密度 | 小爽点密度 | 章节奏 |
|------|-----------|-----------|--------|
| **起点** | 每5000-8000字（2-3章） | 每2000-3000字 | 中等，允许过渡段 |
| **番茄** | 每3000-5000字（1-2章） | 每1000-2000字 | 快，每章都要有看点 |
| **飞卢** | 每2000-3000字（1章） | 每500-1000字 | 极快，几乎每段都有信息量 |

### 密度计算方法

```
目标字数 ÷ 单元故事字数 = 单元故事数量
每个单元故事至少 2-3 个大爽点 + 5-8 个小爽点
每卷至少 1 个卷级大爽点（比单元级更高）
全书至少 2-3 个全书级大爽点
```

## 五、爽点多样性检查

### 检查规则

1. **连续5个大爽点**中至少有 **3种不同类型**
2. **每卷**的爽点类型组合应与相邻卷不同
3. **全书**应覆盖至少 **6-8种** 不同的爽点类型
4. 纯战斗类爽点（B/G/H）占比不应超过 **50%**

### 多样性自检表

| 类型 | 第一卷 | 第二卷 | 第三卷 | ... |
|------|--------|--------|--------|-----|
| A 剧情高峰 | | | | |
| B 高调碾压 | | | | |
| C 低调深沉 | | | | |
| D 扮猪吃虎 | | | | |
| ... | | | | |

每列至少打3个勾，相邻列的类型分布不应完全相同。
