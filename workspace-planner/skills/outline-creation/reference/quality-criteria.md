# 质量审计标准

## 一、审计维度总览

| # | 维度 | 权重 | 检查目标 |
|---|------|------|---------|
| A1 | 爽点密度 | 15% | 大爽点分布是否合理 |
| A2 | 爽点多样性 | 15% | 爽点类型是否重复 |
| A3 | 节奏波浪 | 15% | 是否存在过长"平段" |
| A4 | 冲突类型 | 10% | 相邻卷冲突类型是否重复 |
| A5 | 伏笔链 | 10% | 伏笔是否有收束规划 |
| A6 | 黄金一章 | 15% | 第一章是否合规 |
| A7 | 世界观利用 | 10% | 设定是否被充分利用 |
| A8 | 主角困境 | 10% | 困境是否有多样性 |

---

## 二、各维度详细标准

### A1 爽点密度

**检查方法**：统计每卷的大爽点数量和间隔

| 状态 | 标准 |
|------|------|
| **Pass** | 每3-5章有1个大爽点，无超过5章的爽点空白 |
| **Warning** | 存在1-2处5-8章的爽点空白，但不超过8章 |
| **Fail** | 存在超过8章的爽点空白，或多处5-8章空白 |

**平台调整**：
- 起点：3-5章/大爽点
- 番茄：2-3章/大爽点
- 飞卢：1-2章/大爽点

### A2 爽点多样性

**检查方法**：列出全书所有大爽点，检查类型分布

| 状态 | 标准 |
|------|------|
| **Pass** | 任意连续5个大爽点中至少3种不同类型；全书覆盖≥6种类型 |
| **Warning** | 任意连续5个大爽点中至少2种不同类型；全书覆盖≥4种类型 |
| **Fail** | 连续5个大爽点为同一类型，或全书覆盖<4种类型 |

### A3 节奏波浪

**检查方法**：绘制情绪曲线，检查波峰波谷分布

| 状态 | 标准 |
|------|------|
| **Pass** | 无超过5章的纯"平段"；波峰整体递增；卷末是卷内最高点 |
| **Warning** | 存在1处6-8章平段，或个别卷末不是最高点但有合理解释 |
| **Fail** | 存在超过8章平段，或多处6-8章平段 |

### A4 冲突类型

**检查方法**：检查相邻卷的核心冲突类型

| 状态 | 标准 |
|------|------|
| **Pass** | 相邻卷冲突类型不同 |
| **Warning** | 存在1处相邻卷冲突类型相似，但具体实现方式不同 |
| **Fail** | 存在2处以上相邻卷冲突类型相同且实现方式相似 |

**注意**：冲突类型相似但实现方式完全不同（如"生存"vs"生存"，但一个是物理生存，一个是社会性生存），可以算 Warning 而非 Fail。

### A5 伏笔链

**检查方法**：检查所有长线伏笔是否有明确的收束计划

| 状态 | 标准 |
|------|------|
| **Pass** | 所有长线伏笔有明确的揭晓卷/章；无"埋了忘收"的伏笔 |
| **Warning** | 1-2条长线伏笔收束计划模糊但有大致方向 |
| **Fail** | 3条以上长线伏笔无收束计划，或存在"为伏笔而伏笔"的无效伏笔 |

### A6 黄金一章

**检查方法**：对照黄金一章检查清单逐项检查

| 状态 | 标准 |
|------|------|
| **Pass** | 检查清单全部通过（10/10） |
| **Warning** | 8-9项通过，失败项为非致命项（如"有名字角色3个以内"） |
| **Fail** | <8项通过，或有致命项失败（如"前300字无冲突"） |

**检查清单**（详见 golden-chapter.md）：
1. 前300字内有冲突/危机/悬念
2. 30秒内可理解"发生了什么"
3. 主角身份/处境500字内建立
4. 核心问题1000字内建立
5. 金手指有暗示
6. 有追读钩子
7. 无无意义描写
8. 有名字角色≤3个
9. 删掉第一章后续受影响
10. 结尾有钩子

### A7 世界观利用

**检查方法**：对照世界观设定文件，检查每个核心设定是否有对应剧情冲突

| 状态 | 标准 |
|------|------|
| **Pass** | ≥80%的核心世界观设定有对应剧情冲突 |
| **Warning** | 60-80%的核心设定有对应剧情 |
| **Fail** | <60%的核心设定有对应剧情，或存在大量"设定集式"设定（只写了没用上） |

### A8 主角困境

**检查方法**：检查各卷主角困境模式

| 状态 | 标准 |
|------|------|
| **Pass** | 不存在连续3卷同模式困境 |
| **Warning** | 存在连续2卷同模式困境，但具体实现有差异 |
| **Fail** | 存在连续3卷以上同模式困境（如"被打压→打脸"循环） |

## 四、收敛条件

```
converged = (
    critical_issues.count == 0
    AND warning_issues.count <= 2
    AND overall_score >= 7.0
)
```

### 评分计算

```
overall_score = Σ(dimension.score × dimension.weight) / Σ(weights)
```

各维度评分范围：1-10

| 评级 | 分数 | 含义 |
|------|------|------|
| A | 9.0-10.0 | 优秀，可直接使用 |
| B | 7.0-8.9 | 良好，有小问题需优化 |
| C | 5.0-6.9 | 一般，有明显问题需修复 |
| D | <5.0 | 较差，需要大幅修改 |

## 五、修复建议模板

当审计发现问题时，按以下模板给出修复建议：

```markdown
### Issue: [问题描述]

**维度**: [A1-A8]
**位置**: [具体卷/章]
**严重度**: [critical/warning]

**问题分析**: [为什么这是一个问题]

**修复建议**:
1. [具体建议1]
2. [具体建议2]

**参考方法论**: [指向对应的 reference 文件]
```

### 常见问题的修复方向

| 问题 | 典型修复 |
|------|---------|
| 爽点空白段 | 在空白段中间插入小爽点或K型掐高潮 |
| 爽点类型重复 | 替换为不同类型（如将"高调碾压"改为"低调深沉"） |
| 冲突类型重复 | 重新设计该卷的核心冲突类型 |
| 伏笔无收束 | 补充收束规划，或删除无效伏笔 |
| 第一章不合规 | 按 golden-chapter.md 重写开头 |
| 世界观未利用 | 为闲置设定设计对应剧情冲突 |
| 困境同模式 | 引入新的困境维度（道德/身份/信任等） |
