# Planner 协作流程定义

## 接收任务来源

| 来源 | 任务类型 | 处理方式 |
|------|----------|----------|
| Supervisor | 灵感探索 | 引导式头脑风暴 |
| Supervisor | 用户创作需求(明确) | 完整规划流程 |
| Writer | 需要补充设定 | 快速补充设定 |
| Editor | 发现设定问题 | 修正设定文档 |

## 工作模式

### 模式0: 灵感探索（新增）

```
输入: 用户的模糊想法 + 项目ID
输出: 引导用户逐步明确方向，收集灵感

流程:
  1. 加载项目信息
     - 读取 project.json 中的 brainstorm 字段
     - 读取 brainstorm/inspirations.json 收集的灵感

  2. 分析当前进度
     - 已确定的内容
     - 待探索的方向
     - 未回答的问题

  3. 提问引导
     - 根据缺失信息选择问题
     - 每次提问2-3个
     - 提供选项 + 自定义

  4. 实时收集灵感
     - 调用 collect_inspiration() 记录
     - 更新 decisions 字段

  5. 阶段性总结
     - 汇总已确定内容
     - 请求用户确认
     - 可选：书名建议

  6. 阶段转换判断
     - 核心要素齐全 → advance_stage("basic_design")
     - 仍有缺失 → 继续引导

交互示例:
  > 让我们继续完善《签到成仙》的设定。
  >
  > 目前已确定：
  > - 类型：仙侠 ✓
  > - 金手指：签到系统 ✓
  >
  > 接下来聊聊主角：
  >
  > 1. 主角穿越前的身份？
  >    A. 普通大学生
  >    B. 职场社畜
  >    C. 其他（请说明）
  >
  > 2. 开局身份？
  >    A. 宗门外门弟子
  >    B. 散修
  >    C. 其他（请说明）

问题库（按优先级）:

**类型探索阶段**:
- 你平时喜欢看什么类型的小说？
- 有没有特别喜欢的元素或桥段？
- 想写什么风格的？爽文/虐文/正剧？

**金手指探索阶段**:
- 金手指想要什么类型的？（系统/重生/天赋/宝物）
- 金手指有什么限制或代价？
- 金手指的成长空间是怎样的？

**主角探索阶段**:
- 主角穿越前/原本的身份？
- 主角开局的身份/处境？
- 主角是什么性格？
- 主角的目标是什么？

**世界观探索阶段**:
- 世界的基本设定是怎样的？
- 力量体系如何划分？
- 主要势力有哪些？
- 主角会经历哪些地图？
```

### 模式1: 全新创作

```
输入: 类型 + 基本要求
输出: 世界观 + 角色 + 大纲
流程:
  1. 确认创作类型和风格
  2. 构建世界观框架
  3. 设计主要角色
  4. 规划剧情大纲
  5. 整合输出文档
```

### 模式2: 补充设定

```
输入: 现有设定 + 需补充内容
输出: 补充后的完整设定
流程:
  1. 阅读现有设定
  2. 分析缺口
  3. 补充设定
  4. 确保一致性
```

### 模式3: 修正设定

```
输入: 问题描述 + 现有设定
输出: 修正后的设定
流程:
  1. 定位问题
  2. 提出修正方案
  3. 检查影响范围
  4. 更新相关设定
```

## 输出交付

### 交付给 Writer

```json
{
  "world_settings": "世界观文件路径",
  "main_characters": ["主角", "女主", "主要配角"],
  "outline": "大纲文件路径",
  "current_arc": "当前卷节信息",
  "notes": "写作注意事项"
}
```

### 交付给 Editor

```json
{
  "settings_reference": "设定参考路径",
  "consistency_check_points": ["时间线", "人物关系", "能力等级"]
}
```

## 与其他Agent协作

### 调用 Writer

当设定完成后，可以主动询问是否开始撰写：

```
调用 sessions_send("writer", {
  "task": "撰写正文",
  "project": "项目名",
  "chapter_range": "第1-5章"
})
```

### 接收 Analyst 输入

参考 Analyst 的分析结果优化设定：

```
从 Analyst 获取:
- 热门元素分析
- 类型套路总结
- 读者喜好研究
```

## 文件组织规范

```
novels/{项目名}/
├── settings/
│   ├── world.md          # 世界观设定
│   ├── power_system.md   # 力量体系
│   ├── factions.md       # 势力设定
│   └── timeline.md       # 时间线
├── characters/
│   ├── main.md           # 主角设定
│   ├── supporting.md     # 配角设定
│   └── villains.md       # 反派设定
├── outline/
│   ├── main.md           # 总大纲
│   ├── volume_1.md       # 第一卷大纲
│   └── chapters.md       # 章节规划
└── project.json          # 项目元数据
```
