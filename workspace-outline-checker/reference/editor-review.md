# 编辑审读方法论（多角色审读）

## 核心理念

结构诊断（6框架）回答"骨架是否完整"，编辑审读回答"读者是否会追下去"。
两者互补：结构诊断是 X 光片，编辑审读是临床问诊。

**关键原则**：不同角色有不同的盲区和优先级。单一视角的审读必然遗漏——只有多角色轮审才能逼近全面。

---

## 角色定义

### 角色 1：资深网文编辑

**身份**：从业 8 年以上的商业网文编辑，经手过百部作品，对读者留存数据有直觉级的判断。

**核心关切**：
- 这本书能不能签约？能不能赚钱？
- 读者在前 30 章、100 章、300 章是否会弃书？
- 商业卖点是否清晰且直接？

**审查重点**：

| 维度 | 关注点 | 典型问题 |
|------|--------|---------|
| 开局节奏 | 前 5 章是否有足够钩子让读者留下来 | "第 1 章只有世界观铺垫，没有冲突" |
| 金手指爽感 | 金手指是否在第 1 章就亮？是否足够直接？ | "金手指到 ch8 才用，读者早跑了" |
| 战斗密度 | 每卷是否有至少一场硬战斗高潮？ | "V2 全是政治博弈，0 战斗，追读必崩" |
| 反派压迫感 | 反派是否从第 1 卷就让读者感到威胁？ | "赫连昭到 V4 才有存在感，太晚了" |
| 感情线 | 是否有明确的感情暗示？是否有名场面？ | "200 万字没有感情线，女读者全跑" |
| 篇幅控制 | 每卷篇幅是否合理？是否有压缩空间？ | "V1 用 45 章做入城，读者等不了" |
| 成长系统复杂度 | 读者能否在 5 分钟内说清主角的成长体系？ | "5 条并行成长线，读者记不住" |

**输出风格**：直接、毒舌、用数据说话。"这个开局在起点活不过三章"是正常表达。

---

### 角色 2：结构工程师

**身份**：编剧/故事结构专家，精通 Snyder、Harmon、 McKee 等叙事理论，关注故事骨架的力学合理性。

**核心关切**：
- Midpoint 是否锐利？All Is Lost 是否够痛？
- 每卷是否有完整的情感弧线？
- 伏笔-回收的时间间隔是否合理？

**审查重点**：

| 维度 | 关注点 | 典型问题 |
|------|--------|---------|
| 节拍完整性 | 全书/每卷的 15 个节拍是否齐全 | "全书 Midpoint 模糊——V3 结尾上升，V4 开头继续上升，没有转折" |
| 弧线闭环 | 主角 Want→Need 转变是否有触发事件 | "苏辰从'想活下去'到'守护文明'缺少一个明确的转折点" |
| B Story | 是否有承载主题的关系线 | "没有 B Story——'制度vs个人'主题找不到落点" |
| 张力曲线 | 每卷是否有低谷-高潮的起伏 | "V5 全是上升，45 章没有挫折，读者疲劳" |
| 伏笔管理 | 伏笔设定→回收的间隔是否合理 | "铜匣子在 V1 埋下，V6 才回收，间隔 200 万字" |

**输出风格**：用节拍术语精准定位。"V3 ch49-50 缺 Stakes Raised，需要从 False Victory 变成 Victory But Fear"。

---

### 角色 3：读者代表

**身份**：核心网文读者，年均阅读 2000 万字以上，熟悉各种套路，对"水字数"和"节奏崩"极度敏感。

**核心关切**：
- 我会不会追到第 X 章弃书？
- 哪些地方我会跳着看？哪些地方我会熬夜看？
- 这本书的"上头点"在哪里？

**审查重点**：

| 维度 | 关注点 | 典型问题 |
|------|--------|---------|
| 追读动力 | 每章结尾是否有"不点下一章就睡不着"的钩子 | "ch8-15 是平的，我没有理由继续" |
| 弃读风险点 | 哪些段落最可能让读者关闭页面 | "V4 ch15-20 全是政治描写，连续 6 章没有爽点" |
| 情感投入 | 读者是否关心这些角色的命运 | "老孙头之死要让我哭——现在他只是个功能性 NPC" |
| 期待管理 | 读者的期待是否被及时满足 | "铜臂从 V1 发现到 V5 才融合，读者等得了吗？" |
| 情绪节奏 | 是否有"喘不过气"和"终于松口气"的交替 | "连续 10 章高压，我需要休息" |

**输出风格**：直白、情绪化、用读者语言。"ch10 老孙头死的时候我哭了。但 ch11-20 全是打怪升级，我又困了"。

## 评分体系

### 单维度评分（5 分制）

| 分数 | 含义 |
|------|------|
| 5 | 优秀——无可挑剔，可以作为标杆 |
| 4 | 良好——小问题不影响阅读体验 |
| 3 | 及格——有改进空间但不致命 |
| 2 | 不及格——需要修改 |
| 1 | 致命缺陷——必须重写 |

### 综合评级

| 评级 | 标准 | 下一步 |
|------|------|--------|
| A | 所有维度 ≥ 4，无 P0/P1 | **可以开写** |
| A- | 所有维度 ≥ 3.5，P1 ≤ 2 | 修复 P1 后可开写 |
| B+ | 所有维度 ≥ 3，P0 = 0 | 需要一轮修复 |
| B | 有维度 < 3，或 P0 = 1 | 需要一轮修复 |
| B- | 多个维度 < 3，或 P0 ≥ 2 | 需要两轮修复 |
| C+ | 结构性问题集中 | 需要重大重构 |
| C | 核心设定有缺陷 | 推荐回炉 |

## 审读→修复→复审 流程

### 单轮流程

```
┌─────────────────────────────────────────────────┐
│  1. 角色代入                                    │
│     - 选择角色（编辑/工程师/读者）               │
│     - 阅读完整大纲                               │
│     - 按该角色的审查重点逐项评估                  │
│                                                   │
│  2. 输出审读报告                                  │
│     - 每个维度打分 + 理由                         │
│     - 按严重度排列的问题清单                       │
│     - 综合评级                                    │
│                                                   │
│  3. 修复                                          │
│     - 针对报告中的问题逐项修改大纲                 │
│     - 记录修改内容                                │
│                                                   │
│  4. 复审（同一角色或切换角色）                     │
│     - 验证修复：每个问题是否真正解决               │
│     - 回归检查：修复是否引入新问题                 │
│     - 更新评分                                    │
└─────────────────────────────────────────────────┘
```

### 多角色轮审顺序

**推荐顺序**：编辑先行 → 工程师跟进 → 读者收尾

理由：
1. **编辑先行**：商业问题最致命（开局/金手指/战斗密度），优先解决
2. **工程师跟进**：骨架问题在商业问题修复后再查，避免返工
3. **读者收尾**：以读者视角做最终验证——"修完之后，读者会追吗？"

但也可以根据大纲阶段灵活调整：
- 大纲初稿完成 → 编辑先行
- 骨架已有明显缺陷 → 工程师先行
- 已经过多轮修改 → 读者先行（验证修复效果）

### 收敛规则

1. **最多 3 轮审读**（单角色 3 轮，或 3 角色各 1 轮，或混合）
2. **收敛判定**：
   - 综合评级达到 A 或 A- → 收敛，可开写
   - 第 3 轮仍未达到 A- → 输出"遗留问题清单"，由用户决定是否开写
3. **回归检测**：每轮复审必须检查"修复是否引入新问题"
4. **评分单调性**：如果第 N+1 轮某维度评分低于第 N 轮，必须说明原因

## 实战经验（从 B+→A-→A 的三轮审读中提炼）

### 教训 1：成长系统复杂度是隐形杀手
如果主角有超过 3 条并行成长线，读者会记不住。铜臂从"独立成长线"改为"良性循环的具象化刻度"，5 条线变成 1 条线的 5 个刻度——读者只需要记住一件事。

### 教训 2：反派压迫感必须前置
反派不能在前几卷只是"传说中的威胁"。必须有具体的伤害——杀掉主角认识的人、毁掉主角保护的东西。赫连昭从"遥远威胁"变成"V2 就杀了 37 个三环外居民"，压迫感立竿见影。

### 教训 3：感情线不能靠暗示
200 万字只靠"眼神交汇"和"默契配合"不够。必须有至少一个"名场面"——一个让读者截图分享的瞬间。铜臂握手（"不是手在抖，是我"）同时承载主线+感情+主题，三线合流。

### 教训 4：先打再论，不要只论不打
终极对决不能只是"主角说了更有道理的话"。必须有硬战斗——反派在战场上亲眼看到主角的力量，然后在战斗后的论道中被触动。读者的爽感来自战斗，信服感来自论道，缺一不可。

### 教训 5：修复可能引入新问题
每轮修复后必须用同一维度重新评分。V1 压缩解决了节奏问题，但可能导致入城段铺垫不足——复审必须检查这种回归。
