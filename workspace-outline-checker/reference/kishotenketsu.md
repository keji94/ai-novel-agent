# 起承转合（四幕结构）检查

> 来源：东方叙事传统，网文最自然的节奏单元

## 核心概念

与西方三幕不同，起承转合是四幕结构，强调"转"（twist/转折）的独立地位。网文的自然节奏通常是每 10-15 章一个完整的起承转合循环。

## 四幕检查

### 起（Introduction）— 约前 20% 章节

| 检查项 | 要点 |
|--------|------|
| 开篇钩子 | 第一句话/第一个场景是否抓住注意力？ |
| 核心冲突引入 | 本卷/本段的主冲突是否在 3 章内清晰？ |
| 角色状态确立 | 主角在"起"的阶段是什么状态？ |
| 赌注建立 | 如果不解决冲突，后果是什么？ |

### 承（Development）— 约 20-45% 章节

| 检查项 | 要点 |
|--------|------|
| 冲突深化 | 冲突是否比"起"阶段更复杂？ |
| 爽点释放 | 是否有至少 1-2 个爽点释放（打脸/碾压/突破）？ |
| 新要素引入 | 是否引入了新的角色/势力/信息推进冲突？ |
| 持续张力 | 是否有持续推动读者往前的悬念？ |

### 转（Twist）— 约 45-65% 章节

| 检查项 | 要点 |
|--------|------|
| **意外转折** | 是否有意料之外但情理之中的转折？ |
| 认知升级 | 主角/读者的认知是否被刷新？ |
| 格局变化 | 冲突的性质或规模是否发生了变化？ |
| 失败/挫折 | "转"通常伴随挫折——主角是否遭遇了意想不到的困难？ |

> **"转"是起承转合的灵魂**。没有"转"的故事是"流水账"。每个起承转合单元的"转"的质量决定了叙事的层次感。

### 合（Resolution）— 约 65-100% 章节

| 检查项 | 要点 |
|--------|------|
| 冲突解决 | 本段的主要冲突是否得到了阶段性解决？ |
| 代价体现 | 解决是否付出了代价？（无代价=无重量） |
| 新悬念 | 是否为下一段留下了新的悬念或冲突种子？ |
| 情绪落地 | 情绪是否有释放？读者是否得到满足？ |

## 多层嵌套

起承转合可以多层嵌套：

```
全书级别：
  起 = V1（觉醒+生存）
  承 = V2-V3（成长+开阔）
  转 = V4（挫折+流放）
  合 = V5-V6（重建+决战）

每卷级别：
  每卷 ~45-50 章应有自己的 起→承→转→合

段落级别（可选）：
  每 10-15 章一个微型 起→承→转→合
```

## 每卷检查模板

对每卷填写：

```
卷名：__________ 章数：____

起（ch ___-___）：
  开篇钩子：__________
  核心冲突：__________
  赌注：__________

承（ch ___-___）：
  冲突深化方式：__________
  爽点释放：__________
  新要素：__________

转（ch ___-___）：
  意外转折：__________
  认知升级：__________
  挫折/失败：__________

合（ch ___-___）：
  冲突解决方式：__________
  代价：__________
  新悬念种子：__________

"转"的质量评估：★★★/★★/★
```

## 常见问题

| 问题 | 严重度 | 描述 |
|------|--------|------|
| 整卷无"转" | P0 | 全卷线性发展，没有意外转折 |
| "转"是外部事件 | P2 | 转折仅靠天灾/外力，不是角色选择驱动的 |
| 全书"转"太弱 | P1 | V4（全书转折卷）的挫折不够深 |
| 段落级无起承转合 | P2 | 连续15章+无微型转折 |
| "合"不干净 | P3 | 卷末冲突没解决，也没明确留到下卷 |
| "合"无新悬念 | P2 | 卷末完全封闭，没有推动读者继续的钩子 |
