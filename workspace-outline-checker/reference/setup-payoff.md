# 期待-满足追踪检查

> 伏笔的设定与回收节奏直接影响读者的长期追读粘性。

## 核心概念

- **Setup（设定/伏笔）**：许诺读者"将来会有答案"的信息
- **Payoff（满足/回收）**：兑现之前许诺的答案
- **规则**：每个 Setup 必须有 Payoff，间隔越短读者体验越好

## 检查方法

### 第一步：提取所有 Setup

从大纲中提取所有伏笔/悬念/未解之谜，格式：

```
| # | Setup | 设定位置 | 类型 | 预期回收位置 | 实际回收位置 |
|---|-------|---------|------|------------|------------|
```

### 第二步：分类

| 类型 | 描述 | 回收时效要求 |
|------|------|------------|
| **即时悬念** | 单章内设+解（本章钩子） | 1-3 章内 |
| **段落悬念** | 一个起承转合单元内 | 10-15 章内 |
| **卷级伏笔** | 跨卷的线索 | 当卷末或下卷初 |
| **全书伏笔** | 贯穿多卷的大悬念 | 不超过倒数第二卷 |
| **终局悬念** | 到结局才揭晓的核心谜题 | 必须在结局前揭晓 |

### 第三步：间隔分析

| 间隔 | 评估 | 说明 |
|------|------|------|
| ≤5 章 | 优秀 | 读者还没忘记就回收 |
| 6-15 章 | 合格 | 在一个起承转合单元内 |
| 16-30 章 | 警告 | 需要在回收前有"提醒" |
| 31-50 章 | 危险 | 需要多次提醒，否则读者已忘 |
| >50 章 | **P1** | 极高风险，读者大概率已遗忘 |

### 第四步：提醒检查

对于间隔 >15 章的伏笔，检查是否有"中途提醒"：
- 角色提及
- 相关线索再次出现
- 读者视角的暗示

无提醒的长间隔伏笔 → 升级为 P1

## 常见问题

| 问题 | 严重度 | 描述 |
|------|--------|------|
| 悬念无回收 | P0 | 设了伏笔但从没回收，读者被骗感 |
| 终局悬念拖太久 | P1 | 核心悬念到最后一卷才揭晓，中间缺乏阶段性满足 |
| 回收太集中 | P2 | 大量伏笔在最后几章集中回收，造成信息过载 |
| 无阶段性满足 | P1 | 全书只有大悬念，没有中短期的悬念-满足循环 |
| 回收无惊喜 | P3 | 回收方式太直白，缺乏"原来如此"的惊喜感 |
| 伏笔遗忘 | P2 | 长间隔伏笔无中途提醒，读者已忘 |

## 母亲线索追踪（示例）

以"母亲追查线"这类贯穿全书的悬念为例：

```
Setup: 老孙头"她发现了不该发现的东西"（V1 ch22）
  → 中期满足1: 信残片"封锁线有裂缝"（V1 ch22 立即）
  → 中期满足2: 白老先生"失踪≠死亡"（V2 ch28）
  → 中期满足3: 特别外勤任务代号（V3 ch40）
  → 中期满足4: 外勤日志发现（V4 ch1）
  → 中期满足5: 异族领地内情报人（V5 ch21）
  → 终局满足: 母亲重逢（V6）

评估：每卷至少一个阶段性满足 ✓，间隔合理 ✓
```
