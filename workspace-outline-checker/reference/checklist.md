## 框架互补关系

```
Snyder 15-beat  ── 外部结构骨架（节拍位置）
Harmon Circle   ── 内在弧线骨架（角色心理）
起承转合         ── 每卷节奏骨架（四幕节奏）
                    ↑
              结构层（上面三个）
                    ↓
网文节奏         ── 商业化检查（追读命脉）
期待-满足        ── 悬念管理（长期粘性）
角色+主题        ── 深度保障（口碑基础）
                    ↑
              分析层（上面六个）
                    ↓
编辑审读         ── 多角色实战审读（签约/追读保障）
                  不同角色有不同盲区
                  多轮迭代直到收敛
```

## 建议执行顺序

### Phase 1：结构诊断（大纲初稿完成后）
1. Snyder 15-beat → 确认骨架完整
2. 网文节奏 → 确认爽点密度
3. 期待-满足 → 确认伏笔节奏
4. 起承转合 → 逐卷检查节奏
5. Harmon Circle → 检查主角弧线
6. 角色+主题 → 检查深度

### Phase 2：多角色编辑审读（结构诊断修复后）
7. 编辑审读 → 多轮审读（编辑→工程师→读者）
   - 推荐：资深网文编辑先行 → 结构工程师跟进 → 读者代表收尾
   - 每轮审读后修复，然后复审
   - 综合评级达到 A 或 A- 时收敛

### Phase 3：每卷写作前
- 起承转合 + 网文节奏 → 聚焦该卷
