## 二、Phase 3 改进详情

### 2.1 Detector Agent (AI 痕迹检测器)

**功能**: 检测文本中的 AI 生成痕迹

**检测维度**:

#### 确定性规则（11条，零成本）

| 规则 | 检测内容 | 权重 |
|------|----------|------|
| 禁止句式 | 「不是...而是...」 | 10 |
| 禁止破折号 | 「——」 | 8 |
| 转折词密度 | 仿佛/忽然/竟然 过多 | 5 |
| 高疲劳词 | 题材疲劳词超频 | 5 |
| 元叙事 | 编剧旁白式表述 | 10 |
| 报告术语 | 分析框架术语 | 5 |
| 作者说教 | 显然/不言而喻 | 5 |
| 集体反应 | 「全场震惊」套话 | 5 |
| 连续了字 | ≥6 句连续「了」 | 3 |
| 段落过长 | ≥2 段超 300 字 | 2 |
| 列表式结构 | 首先/其次/最后 | 8 |

#### 统计特征检测

| 特征 | 阈值 |
|------|------|
| TTR（词汇多样性） | ≥ 0.05 |
| 句长标准差 | ≥ 10 |
| 主动句比例 | ≥ 50% |

**评分系统**:
- 90-100: 极低 AI 痕迹，通过
- 80-89: 低 AI 痕迹，通过
- 70-79: 中等 AI 痕迹，建议修订
- 60-69: 较高 AI 痕迹，需要修订
- <60: 高 AI 痕迹，建议重写

### 2.2 导出工具

**功能**: 支持多格式导出

**支持格式**:

| 格式 | 说明 | 依赖 |
|------|------|------|
| TXT | 纯文本格式 | 无 |
| Markdown | Markdown 格式，含目录 | 无 |
| EPUB | 电子书格式，支持阅读器 | pandoc |

**使用方法**:
```bash
# 导出 TXT
./scripts/export.sh txt 仙道长生

# 导出 Markdown
./scripts/export.sh md 仙道长生

# 导出 EPUB
./scripts/export.sh epub 仙道长生

# 只导出已审核章节
./scripts/export.sh epub 仙道长生 --approved-only

# 指定章节范围
./scripts/export.sh txt 仙道长生 --start 10 --end 50
```

## 六、核心竞争力

相比 InkOS，我们的独特优势：

### 1. 架构优势
- **Supervisor 模式**: 灵活的任务路由，支持并行/串行/多轮对话
- **OpenClaw 生态**: 天然支持多渠道（CLI、飞书、Webchat）
- **Subagent 隔离**: 每个 Agent 独立会话，上下文不污染
- **易于扩展**: 新增 Agent 只需创建 workspace + 配置

### 2. 功能完备
- ✅ 两阶段写作 + 7 真相文件
- ✅ 33 维度审计 + 写后验证器
- ✅ 章节导入 + 文风仿写
- ✅ AI 痕迹检测 + EPUB 导出

### 3. 零外部依赖
- 确定性规则检测零 LLM 成本
- 导出工具纯 shell 脚本
- 可在无网络环境运行
